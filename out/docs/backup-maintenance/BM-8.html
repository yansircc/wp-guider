更新 WordPress 与插件

备份与维护 L2

本文同时覆盖: BM-8, BM-9

📖 What — 这是什么

WordPress 核心、插件和主题都需要定期更新。更新通常包含安全修复、bug 修复和新功能。WordPress 后台的"更新"页面集中显示所有可用更新。

更新分为两类：

  • 小版本更新（如 6.4.1 → 6.4.2）— 安全/bug 修复，WordPress 默认自动执行
  • 大版本更新（如 6.4 → 6.5）— 包含新功能，需要手动确认

🎯 Why — 为什么要做

过期的 WordPress 和插件是被黑客入侵的头号原因。已知漏洞一旦公开，攻击者会立即开始大规模扫描未更新的站点。

更新 WordPress 不是"有空再做"的事，而是安全维护的核心任务。

🔧 How — 怎么做

更新前：备份

# 用 WP Reset 创建快照（训练环境）
# 或用 UpdraftPlus 做完整备份（生产环境）

更新 WordPress 核心

# 查看当前版本
locwp wp wp-train -- core version

# 检查是否有更新
locwp wp wp-train -- core check-update

# 执行更新
locwp wp wp-train -- core update

更新插件

# 查看需要更新的插件
locwp wp wp-train -- plugin list --update=available

# 更新所有插件
locwp wp wp-train -- plugin update --all

# 更新指定插件
locwp wp wp-train -- plugin update wordfence

更新主题

# 更新所有主题
locwp wp wp-train -- theme update --all

更新后：验证

  1. 检查前台页面是否正常显示
  2. 检查后台功能是否正常
  3. 检查关键功能（表单、支付等）
  4. 查看 debug.log 是否有新错误

⚠️ 常见问题与注意事项

  • 先更新插件，再更新核心——这样出问题时更容易定位原因
  • 不要在流量高峰期执行更新
  • 更新可能导致插件兼容性问题，特别是大版本更新时
  • 如果更新后站点崩溃，用备份恢复，然后逐个更新排查是哪个造成问题
  • 长期不维护的插件（超过 1 年没更新）应该考虑替换，而不是继续使用
  • 生产环境建议先在测试站点上验证更新后再应用到正式站点

wp-cron 检查

备份与维护 L3

📖 What — 这是什么

wp-cron 是 WordPress 内建的定时任务系统。它负责执行计划发布文章、自动备份、插件定时任务等周期性工作。

关键问题：wp-cron 不是真正的 cron。它是"伪 cron"——只有在有人访问站点时才会检查并执行到期的任务。如果没人访问，任务就不会执行。

🎯 Why — 为什么要做

wp-cron 的"伪 cron"机制会导致几个问题：

  • 定时发布的文章可能延迟发布（没有访客触发 cron）
  • 自动备份不按时执行
  • 每次页面加载都要检查 cron 队列，增加响应时间
  • 高流量站点可能触发 cron 过于频繁，导致任务重复执行

理解 wp-cron 的工作原理，才能判断是否需要用系统 cron 替代。

🔧 How — 怎么做

查看 cron 事件列表

# 列出所有计划任务
locwp wp wp-train -- cron event list

输出会显示每个任务的钩子名、下次执行时间和执行频率。

手动触发 cron

# 执行所有到期的 cron 事件
locwp wp wp-train -- cron event run --due-now

检查 wp-cron 状态

# 检查 DISABLE_WP_CRON 常量
locwp wp wp-train -- config get DISABLE_WP_CRON 2>/dev/null || echo "未设置（wp-cron 启用中）"

常见的 cron 任务

  • wp_version_check — 检查 WordPress 更新
  • wp_update_plugins — 检查插件更新
  • wp_scheduled_delete — 清理回收站中的内容
  • updraft_backup — UpdraftPlus 自动备份（如已配置）

⚠️ 常见问题与注意事项

  • wp-cron 依赖页面访问触发，低流量站点可能出现任务积压
  • 不要手动删除 cron 事件，除非你确定知道它是什么——很多插件依赖 cron 工作
  • 如果 cron 事件列表中有大量"过期"事件（过了执行时间但未执行），说明 wp-cron 可能被禁用或有问题
  • 某些安全插件或缓存插件可能会干扰 wp-cron 的正常执行
  • 对于需要准时执行的任务（如定时发布），建议配置系统 cron

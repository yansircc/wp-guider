系统 cron 替换 wp-cron

备份与维护 L4

📖 What — 这是什么

用 Linux 系统的 cron（或 macOS 的 launchd）定时调用 WordPress 的 cron 端点，替代 WordPress 内建的伪 cron 机制。

这样做的效果：wp-cron 不再依赖页面访问触发，而是由操作系统准时触发。

🎯 Why — 为什么要做

对于生产环境，系统 cron 替代 wp-cron 有三个优势：

  1. 准时执行 — 不依赖访客触发，定时任务准时执行
  2. 减少前端负担 — 普通页面请求不再需要检查 cron 队列
  3. 避免重复执行 — 高流量站点不会因为并发访问导致任务多次触发

🔧 How — 怎么做

步骤一：禁用 WordPress 内建 wp-cron

locwp wp wp-train -- config set DISABLE_WP_CRON true --raw

这会在 wp-config.php 中添加：

define( 'DISABLE_WP_CRON', true );

步骤二：配置系统 cron（Linux 生产服务器）

# 编辑 crontab
crontab -e

# 每 15 分钟触发一次 WordPress cron
*/15 * * * * wget -q -O /dev/null https://your-site.com/wp-cron.php?doing_wp_cron >/dev/null 2>&1

或者用 curl：

*/15 * * * * curl -s https://your-site.com/wp-cron.php?doing_wp_cron >/dev/null 2>&1

或者用 WP-CLI（推荐，不依赖 HTTP）：

*/15 * * * * cd /path/to/wordpress && wp cron event run --due-now >/dev/null 2>&1

Local 环境说明

在 Local (localWP) 训练环境中，不需要配置系统 cron。理解原理即可。可以用 WP-CLI 手动触发来模拟：

locwp wp wp-train -- cron event run --due-now

验证

# 确认 DISABLE_WP_CRON 已设置
locwp wp wp-train -- config get DISABLE_WP_CRON

# 手动触发后检查事件是否被执行
locwp wp wp-train -- cron event list

⚠️ 常见问题与注意事项

  • 禁用 wp-cron 后如果忘记配置系统 cron，所有定时任务都不会执行（定时发布、自动备份等）
  • 频率建议：每 15 分钟一次是合理的平衡点。不需要每分钟执行
  • 使用 WP-CLI 方式比 HTTP 方式更可靠，不受防火墙/WAF 限制
  • 多站点环境需要为每个站点配置独立的 cron 任务
  • 共享主机可能不允许配置 crontab，此时只能保留 wp-cron

WP Reset 快照管理

备份与维护 L2 NEW

本文同时覆盖: BM-5, BM-6

📖 What — 这是什么

WP Reset 插件的"快照"（Snapshot）功能可以在几秒内保存和恢复 WordPress 数据库状态。它只保存数据库，不包含文件——这使得操作极其轻量和快速。

快照就像游戏中的存档点：做操作前存一个，出问题了直接读档。

🎯 Why — 为什么要做

在训练环境中，你需要频繁尝试各种操作——安装插件、修改设置、调整主题。快照让你可以大胆尝试，因为随时可以回到之前的状态。

与 UpdraftPlus 的完整备份相比，WP Reset 快照创建和恢复都只需几秒，非常适合"尝试 → 出错 → 恢复 → 重试"的学习循环。

🔧 How — 怎么做

安装 WP Reset

locwp wp wp-train -- plugin install wp-reset --activate

创建快照

  1. 进入 工具 → WP Reset → Snapshots 标签
  2. 输入快照描述（如"安装 Wordfence 前"、"初始干净状态"）
  3. 点击 Create Snapshot

快照创建通常在 1-3 秒内完成。

恢复快照

  1. 在快照列表中找到目标快照
  2. 点击 Restore snapshot
  3. 在确认弹窗中输入 restore 并确认

恢复同样只需几秒。恢复后页面会自动刷新。

管理快照

  • 每个快照可以下载为 SQL 文件
  • 可以比较两个快照之间的差异
  • 不需要的快照可以删除以节省空间

最佳实践

建议在以下时机创建快照：

  • 开始新的训练课程前
  • 安装新插件/主题前
  • 修改重要设置前
  • 任何"我不确定会不会出问题"的操作前

⚠️ 常见问题与注意事项

  • 快照只包含数据库，不包含文件。如果你手动修改了 wp-config.php 或上传了文件，恢复快照不会回滚这些更改
  • 恢复快照会覆盖当前数据库中的所有数据，操作不可逆
  • 快照存储在数据库中，过多的快照会增加数据库体积
  • WP Reset 还有一个"Reset"（重置）功能，会删除所有内容恢复到全新安装状态——这跟快照恢复完全不同，使用时要注意区分
  • 免费版最多保存 2-3 个快照，训练环境足够使用

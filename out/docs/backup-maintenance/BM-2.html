恢复备份

backup-maintenance L3

📖 What — 这是什么

从 UpdraftPlus 备份中恢复站点到之前的状态。恢复操作可以选择性执行：只恢复数据库、只恢复插件，或者全部恢复。

恢复会用备份中的数据覆盖当前数据，这是不可逆操作——除非你在恢复前又做了一次备份。

🎯 Why — 为什么要做

备份的价值只有在恢复时才能体现。更新出错导致白屏、被黑客入侵、误操作删除了关键内容——这些场景都需要恢复能力。

如果你不练习恢复，出事时可能发现：备份文件损坏、恢复步骤不熟、关键数据根本没包含在备份中。在训练环境中练习恢复是零风险的。

🔧 How — 怎么做

从现有备份恢复

  1. 进入 设置 → UpdraftPlus Backups
  2. 在 现有备份（Existing Backups）列表中找到目标备份
  3. 点击 还原（Restore）按钮
  4. 勾选需要恢复的组件：数据库、插件、主题、上传文件、其他
  5. 点击 Next，确认执行恢复
  6. 等待恢复完成，页面会显示进度条
恢复会覆盖当前数据，操作前务必确认。建议在恢复前先做一次当前状态的备份（UpdraftPlus 恢复时会提示是否自动备份）。

从上传的备份文件恢复

如果备份文件不在当前站点上（例如从其他地方拷贝过来的）：

  1. 点击 上传备份文件
  2. 拖拽或选择备份的 zip/gz 文件
  3. 上传完成后，备份会出现在列表中
  4. 按上述步骤执行恢复

⚠️ 常见问题与注意事项

  • 恢复数据库会覆盖所有文章、页面、设置和用户数据，请确认后再操作
  • 恢复后建议重新保存固定链接：设置 → 固定链接 → 保存更改
  • 如果恢复过程中断，站点可能处于不一致状态——重新执行恢复即可修复
  • 大文件恢复可能超时（特别是共享主机），可以尝试分组件逐步恢复
  • 如果 WordPress 后台进不去，UpdraftPlus 的恢复功能也无法使用。这时需要通过 FTP 手动恢复文件 + phpMyAdmin 导入数据库
  • 恢复后仔细检查前后台功能和媒体文件是否完整

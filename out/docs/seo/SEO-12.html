CPT 单篇 + 归档页的 SEO 策略

SEO L4 NEW

📖 What — 这是什么

自定义文章类型有两种页面：单篇页（如 /product/iphone/）和归档页（如 /products/）。需要根据业务场景决定哪些页面该被收录（index）、哪些不该（noindex）。

🎯 Why — 为什么要做

  • 不是所有 CPT 页面都适合出现在搜索结果中
  • 错误的 index 策略会导致低质量页面稀释网站 SEO 权重
  • 归档页和单篇页可能存在内容重复问题

🔧 How — 决策框架

单篇页（Single）

  • index（收录）：内容丰富、有独立价值的页面 — 产品详情页、案例展示页、FAQ 单篇
  • noindex（不收录）：内容薄弱或仅用于内部管理的页面 — 如 CPT 只是用于后台管理数据而非前端展示

归档页（Archive）

  • index：归档页有独立价值（如产品列表页，用户会搜索「XX品牌产品」）
  • noindex：归档页内容与单篇页高度重复，或内容太少（只有标题列表）

常见组合策略

CPT 类型 单篇 归档 说明
产品 index index 两者都有搜索价值
团队成员 index noindex 单篇有价值，列表页意义不大
弹窗/横幅 noindex noindex 纯内部管理用途
FAQ index index 常见问题有搜索流量
原则：如果用户可能直接搜索到这个页面 → index；如果页面只是为网站内部组织服务 → noindex。

⚠️ 常见问题与注意事项

  • 归档页需要先开启：ACF 创建 CPT 时需在高级 → URL → 归档中开启归档，否则根本没有归档页可言
  • 分类法归档也要考虑：自定义分类法的归档页（如 /product-category/xxx/）也需要做 index/noindex 决策
  • 分页问题：归档页的第 2、3...N 页建议 noindex（SEO 插件通常自动处理）

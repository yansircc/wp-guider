生成 Application Password

用户管理 L3

📖 What — 这是什么

Application Password（应用密码）是 WordPress 5.6 引入的功能，允许为第三方应用或脚本生成专用密码，通过 REST API 访问 WordPress，而无需使用主密码。

每个应用密码是独立的，可以随时吊销，不影响用户的主登录密码。这是 WordPress REST API 认证的推荐方式。

🎯 Why — 为什么要做

在 WP Guider 训练系统中，Claude Code 通过 REST API 验证教练的操作结果。Application Password 让 AI agent 能够安全地读取站点数据，用于自动化检查。

相比直接使用登录密码，Application Password 更安全：权限范围相同但可以单独吊销，泄露后不影响主密码。

🔧 How — 怎么做

  1. 登录 WordPress 后台
  2. 进入 用户 → 我的个人资料（Users → Profile）
  3. 滚动到页面底部，找到 Application Passwords 区域
  4. 在 New Application Password Name 中输入描述名称（如 "WP Guider Agent"）
  5. 点击 Add New Application Password
  6. 系统生成一个密码（格式如 xxxx xxxx xxxx xxxx xxxx xxxx），立即复制保存
密码只会显示一次！关闭页面后无法再次查看。如果忘记了，只能删除重新生成。

使用方式

通过 HTTP Basic Auth 调用 REST API：

curl -u "username:xxxx xxxx xxxx xxxx xxxx xxxx" \
  https://your-site.com/wp-json/wp/v2/posts

⚠️ 常见问题与注意事项

  • Application Password 要求站点使用 HTTPS，HTTP 站点无法使用
  • 如果看不到 Application Passwords 区域，可能是被安全插件禁用了（如 iThemes Security）
  • 密码中的空格可以保留也可以去掉，都能正常使用
  • 每个 Application Password 的权限与其关联用户相同，管理员的应用密码拥有管理员权限
  • 定期检查不再使用的应用密码并 Revoke（吊销）

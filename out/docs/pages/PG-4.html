修改页面 slug

页面管理 L2

📖 What — 这是什么

Slug 是页面 URL 中标识该页面的部分。例如 URL https://example.com/about-us/ 中，about-us 就是 slug。WordPress 会根据页面标题自动生成 slug，但中文标题会产生编码混乱的 URL。

修改 slug 是指手动设置一个简洁、有意义的英文 URL 标识符，让页面地址更友好、更专业。

🎯 Why — 为什么要做

中文标题生成的 slug 会被 URL 编码为一串 %E4%B8%AD%E6%96%87 这样的乱码，既不美观也不利于 SEO。手动设置英文 slug 可以让 URL 简洁可读，如 /about/ 而非 /%E5%85%B3%E4%BA%8E/。

搜索引擎偏好包含关键词的短 URL。一个清晰的 slug 既能提升搜索排名，也能让用户在分享链接时更有信心。

🔧 How — 怎么做

  1. 进入 页面 → 所有页面，找到目标页面并点击编辑
  2. 在区块编辑器中，点击页面标题上方的 URL / 固定链接区域
  3. 在弹出的编辑框中修改 slug 为英文，例如将「关于我们」改为 about
  4. slug 中用连字符 - 分隔单词，如 contact-us
  5. 点击 更新 保存修改

替代方法：在「所有页面」列表中，将鼠标悬停在页面名称上，点击「快速编辑」，可以直接修改 slug 而无需进入编辑器。

⚠️ 常见问题与注意事项

  • 已发布页面修改 slug 会改变 URL：旧的 URL 将返回 404。如果该页面已被外部链接引用或被搜索引擎收录，需要设置 301 重定向
  • slug 命名规范：只使用小写英文字母、数字和连字符，避免使用下划线、特殊字符和中文
  • slug 重复：如果输入的 slug 已被其他页面/文章使用，WordPress 会自动追加 -2，注意检查
  • 草稿状态下可以随意修改：页面发布前修改 slug 没有任何风险，建议在发布前就设置好

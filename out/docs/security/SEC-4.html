禁用文件编辑

security L3

📖 What — 这是什么

WordPress 后台自带"主题编辑器"（外观 → 主题文件编辑器）和"插件编辑器"（插件 → 插件文件编辑器），允许管理员直接在浏览器中修改 PHP、CSS、JS 等源文件。

通过在 wp-config.php 中设置 DISALLOW_FILE_EDIT 常量为 true，可以彻底禁用这两个编辑器。设置后，后台菜单中对应的入口会消失，直接访问编辑器 URL 也会被拒绝。

🎯 Why — 为什么要做

后台文件编辑器是一个巨大的安全隐患。如果攻击者通过钓鱼、密码泄露或其他方式获取了管理员账号，他们可以直接通过编辑器向主题或插件文件中注入恶意代码——不需要 FTP 权限，不需要 SSH 访问，只需要一个浏览器。

即使不考虑安全问题，后台编辑器也容易造成误操作。一个手误就可能导致白屏（PHP fatal error），而且没有版本控制，无法回滚。

禁用后台编辑器只是关闭了一个攻击入口，不影响通过 FTP、SSH 或本地文件系统直接修改文件。真正的文件保护需要配合正确的文件权限设置。

🔧 How — 怎么做

1. 通过 WP-CLI 设置常量

locwp wp wp-train -- config set DISALLOW_FILE_EDIT true --type=constant

这会在 wp-config.php 中添加一行：

define( 'DISALLOW_FILE_EDIT', true );

2. 验证设置生效

# 确认常量值
locwp wp wp-train -- config get DISALLOW_FILE_EDIT

应返回 true。

3. 在后台验证

  1. 登录后台，进入 外观 菜单 — "主题文件编辑器" 应该已消失
  2. 进入 插件 菜单 — "插件文件编辑器" 应该已消失
  3. 直接访问 /wp-admin/theme-editor.php — 应显示"抱歉，您不能编辑此站点的主题"

⚠️ 常见问题与注意事项

DISALLOW_FILE_EDIT vs DISALLOW_FILE_MODS

WordPress 有两个相关但不同的常量：

  • DISALLOW_FILE_EDIT — 仅禁用后台编辑器。插件和主题仍然可以正常安装、更新、删除
  • DISALLOW_FILE_MODS — 禁用一切文件修改操作，包括安装/更新/删除插件和主题、后台编辑器、自动更新。这个常量包含了 DISALLOW_FILE_EDIT 的效果
训练环境中不要启用 DISALLOW_FILE_MODS，否则你将无法通过后台安装任何插件或主题，严重影响学习进度。

其他注意事项

  • 这个设置是写在 wp-config.php 中的，不是数据库选项，因此不能通过后台修改
  • 如需恢复编辑器，用 WP-CLI 删除常量：locwp wp wp-train -- config delete DISALLOW_FILE_EDIT
  • 生产环境中，DISALLOW_FILE_EDIT 应该是标配。所有代码变更应通过版本控制（Git）和部署流程完成
  • 部分安全插件（如 Sucuri、iThemes Security）也提供禁用编辑器的选项，效果相同但本质上都是设置这个常量

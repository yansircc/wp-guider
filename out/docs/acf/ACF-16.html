图片 / 文件字段

ACF L2

📖 What — 这是什么

ACF 的媒体类字段让你为文章关联图片和文件附件：

  • 图片（Image）：从媒体库选择或上传图片，支持预览
  • 文件（File）：上传任意文件类型（PDF、文档、压缩包等）

与直接在正文插入图片不同，ACF 图片字段是结构化数据，可以在模板的任意位置精确调用。

🎯 Why — 为什么要做

  • 产品主图、团队头像、案例封面等需要独立于正文的图片字段
  • 产品规格书、下载资料等需要文件字段
  • 结构化的图片/文件字段可以在不同模板位置调用，比正文更灵活

🔧 How — 怎么做

图片字段设置

  1. 添加字段时选择类型「图片」
  2. 关键配置：
    • 返回格式：
      • 图片数组（推荐）— 返回完整信息（URL、尺寸、alt 等）
      • 图片 URL — 只返回 URL 字符串
      • 图片 ID — 只返回附件 ID
    • 预览尺寸：编辑界面显示的缩略图大小
    • 限制文件类型：如 jpg, png, webp
    • 尺寸限制：可设置最小/最大宽高

文件字段设置

  1. 添加字段时选择类型「文件」
  2. 关键配置：
    • 返回格式：文件数组、URL 或 ID
    • 限制文件类型：如 pdf, doc, docx
    • 文件大小限制：避免上传过大文件

实际例子

产品目录中添加产品主图：

  • 字段标签：产品主图
  • 字段类型：图片
  • 返回格式：图片数组
  • 限制类型：jpg, png, webp
  • 最小尺寸：800×600

⚠️ 常见问题与注意事项

  • 返回格式选择：如果前端需要 alt 文字和多种尺寸，选「图片数组」；如果只需要 URL 嵌入，选「图片 URL」
  • 上传限制：WordPress 默认上传大小受服务器 PHP 配置限制（upload_max_filesize）
  • 图片优化：建议上传前压缩图片，ACF 不会自动压缩
  • 必填图片：设为必填后，编辑者必须上传图片才能发布文章

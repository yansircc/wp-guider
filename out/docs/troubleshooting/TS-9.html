综合故障注入 — 数据库错误

troubleshooting L5

📖 What — 这是什么

"Error establishing a database connection" 是 WordPress 最令人恐慌的错误之一。它意味着 WordPress 无法连接到数据库，整个站点——前台和后台——都完全瘫痪。

在故障注入训练中，系统会故意破坏数据库连接（如修改 DB_NAME/DB_PASSWORD），你需要诊断并恢复。

🎯 Why — 为什么要做

数据库连接错误的常见原因：

  • wp-config.php 中的数据库凭据被误改
  • 数据库服务器宕机或重启
  • 数据库用户权限被修改
  • 数据库被删除或损坏
  • 迁移后数据库配置未更新

这类故障完全无法通过后台修复，只能通过命令行或文件级别操作。

🔧 How — 怎么做

诊断流程

  1. 确认错误类型 
    # 测试数据库连接
locwp wp wp-train -- db check 2>&1

    如果报连接错误，说明凭据或服务有问题。

  2. 检查数据库配置 
    # 查看当前配置
locwp wp wp-train -- config get DB_NAME
locwp wp wp-train -- config get DB_USER
locwp wp wp-train -- config get DB_HOST
  3. 检查数据库服务 
    # 数据库服务是否在运行？
# Local 环境可以在 Local app 中查看站点状态

修复方法

# 如果是 DB_NAME 被改错
locwp wp wp-train -- config set DB_NAME "local"

# 如果是密码被改错
locwp wp wp-train -- config set DB_PASSWORD "root"

# 修复后验证
locwp wp wp-train -- db check

数据库损坏时

# WordPress 内置修复工具
# 在 wp-config.php 中添加：
# define('WP_ALLOW_REPAIR', true);
# 然后访问 /wp-admin/maint/repair.php

# 或用 WP-CLI
locwp wp wp-train -- db repair

⚠️ 常见问题与注意事项

  • Local 环境的默认数据库凭据通常是：DB_NAME=local，DB_USER=root，DB_PASSWORD=root
  • 如果数据库真的被删除了，只能从备份恢复
  • 使用 WP_ALLOW_REPAIR 修复后务必删除该常量——它允许任何人访问修复页面
  • 数据库密码等敏感信息不要在日志或聊天中明文分享
  • 这个故障场景再次证明了备份的重要性——没有备份的数据库丢失是灾难性的

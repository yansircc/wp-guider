修复错误 siteurl（排障版）

troubleshooting L5

📖 What — 这是什么

这是一个故障注入场景：siteurl 或 home 被故意改成错误的值，模拟真实环境中误操作导致站点完全无法访问的情况。

症状：浏览器加载超时或重定向循环，后台登录页面无法打开，所有 CSS/JS 资源加载失败。

🎯 Why — 为什么要做

siteurl 错误是 WordPress 最常见的"自锁"故障之一。在实际工作中，这可能由以下原因触发：

  • 在后台"常规设置"中误改了站点地址
  • 数据库迁移时 URL 替换不完整
  • SSL 切换时忘记同步更新 URL

在安全的训练环境中练习修复，比在生产环境手忙脚乱强一百倍。

🔧 How — 怎么做

诊断步骤

# 1. 确认 siteurl 的当前值
locwp wp wp-train -- option get siteurl
locwp wp wp-train -- option get home

# 2. 对比正确的 URL（Local 环境通常是）
# http://wp-train.local

修复步骤

# 方法一：WP-CLI（推荐）
locwp wp wp-train -- option update siteurl "http://wp-train.local"
locwp wp wp-train -- option update home "http://wp-train.local"

# 方法二：SQL 直接修改
locwp wp wp-train -- db query "UPDATE wp_options SET option_value='http://wp-train.local' WHERE option_name IN ('siteurl','home');"

# 方法三：wp-config.php 硬编码（临时方案）
# 在 wp-config.php 中添加：
# define('WP_HOME', 'http://wp-train.local');
# define('WP_SITEURL', 'http://wp-train.local');

修复后清理

# 清除对象缓存
locwp wp wp-train -- cache flush

# 刷新固定链接
locwp wp wp-train -- rewrite flush

# 如果数据库中还有旧 URL 引用
locwp wp wp-train -- search-replace "http://wrong-url.local" "http://wp-train.local" --dry-run
locwp wp wp-train -- search-replace "http://wrong-url.local" "http://wp-train.local"

⚠️ 常见问题与注意事项

  • siteurl 和 home 必须同时修复，只改一个可能导致混合内容问题
  • URL 末尾不加斜杠：http://wp-train.local 而非 http://wp-train.local/
  • 如果使用了 wp-config.php 硬编码方式修复，修复后记得删除那两行，否则后台设置会变灰不可编辑
  • search-replace 命令先用 --dry-run 预览影响范围，确认无误再执行
  • 这是排障训练中的高频考题——确保你能在 2 分钟内完成修复

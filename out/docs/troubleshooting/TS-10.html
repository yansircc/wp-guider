综合故障注入 — 500 Internal Server Error

troubleshooting L5

📖 What — 这是什么

HTTP 500 Internal Server Error 是一个通用的服务器端错误。它表示服务器在处理请求时遇到了意外情况，但不告诉你具体是什么。

在 WordPress 中，500 错误通常由以下原因引起：.htaccess 文件损坏、PHP 内存耗尽、插件/主题代码错误、或服务器配置问题。

🎯 Why — 为什么要做

500 错误的棘手之处在于它的"通用性"——可能的原因太多，错误信息太少。需要系统性地逐个排除。

与白屏不同，500 错误通常在 Web 服务器级别就被截获了，WordPress 本身可能还没机会运行。因此排障需要从服务器日志开始，而不仅仅是 debug.log。

🔧 How — 怎么做

系统性排障流程

  1. 查看服务器错误日志 
    # Apache 错误日志
tail -30 /var/log/apache2/error.log

# Nginx 错误日志
tail -30 /var/log/nginx/error.log

# Local 环境日志位置因系统而异
  2. 查看 WordPress 调试日志 
    locwp wp wp-train -- config set WP_DEBUG true --raw
locwp wp wp-train -- config set WP_DEBUG_LOG true --raw
tail -30 wp-content/debug.log
  3. 检查 .htaccess（Apache 环境） 
    # 重命名 .htaccess 排除
mv .htaccess .htaccess.bak

# 重新生成
locwp wp wp-train -- rewrite flush
  4. 检查 PHP 内存 
    locwp wp wp-train -- config set WP_MEMORY_LIMIT 256M --raw
  5. 标准排查 — 停用插件 → 切换主题 → 检查 wp-config.php

⚠️ 常见问题与注意事项

  • 500 错误可能是间歇性的——有时正常有时报错。这通常指向内存或资源限制问题
  • 如果只有后台报 500 而前台正常，重点排查后台相关的插件功能
  • Local 环境使用 Nginx，不存在 .htaccess 问题
  • PHP 版本不兼容也可能导致 500 错误——特别是更新 PHP 版本后
  • 排障时注意区分是"所有页面都 500"还是"只有特定页面 500"——范围不同，原因不同

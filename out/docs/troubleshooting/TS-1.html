开启 WP_DEBUG 与查看 debug.log

troubleshooting L3

本文同时覆盖: TS-1, TS-2

📖 What — 这是什么

WordPress 内置了调试模式，通过 wp-config.php 中的三个常量控制：

  • WP_DEBUG — 总开关，启用后 PHP 错误和警告不再被隐藏
  • WP_DEBUG_LOG — 将错误信息写入 wp-content/debug.log 文件
  • WP_DEBUG_DISPLAY — 是否在页面上直接显示错误

debug.log 是排障的第一手信息来源——PHP 致命错误、弃用警告、插件冲突线索都会记录在这里。

🎯 Why — 为什么要做

WordPress 默认隐藏所有错误信息，遇到问题时你只会看到白屏或含糊的错误页面。开启调试模式后，真正的错误原因会被记录下来。

这是系统性排障的第一步：先获取错误信息，再定位根因。没有 debug.log，排障就是盲人摸象。

🔧 How — 怎么做

开启调试模式（推荐配置）

# 开启调试总开关
locwp wp wp-train -- config set WP_DEBUG true --raw

# 记录到日志文件
locwp wp wp-train -- config set WP_DEBUG_LOG true --raw

# 不在页面上显示（安全考虑）
locwp wp wp-train -- config set WP_DEBUG_DISPLAY false --raw

查看 debug.log

# 查看最新的 50 行
tail -50 wp-content/debug.log

# 实时监控日志（新错误会自动刷新）
tail -f wp-content/debug.log

# 搜索特定错误
grep "Fatal error" wp-content/debug.log
grep "Warning" wp-content/debug.log

读懂日志条目

典型的日志条目格式：

[30-Mar-2026 10:15:32 UTC] PHP Fatal error:
  Uncaught Error: Call to undefined function my_func()
  in /app/public/wp-content/plugins/bad-plugin/main.php on line 42

关键信息：

  • 错误类型 — Fatal error（致命）、Warning（警告）、Notice（提示）
  • 错误描述 — 调用了未定义的函数 my_func()
  • 文件位置 — plugins/bad-plugin/main.php 第 42 行

关闭调试模式

locwp wp wp-train -- config set WP_DEBUG false --raw

⚠️ 常见问题与注意事项

  • 生产环境务必关闭 WP_DEBUG_DISPLAY，防止向访客泄露系统信息
  • debug.log 会持续增长，排障完成后记得清理或关闭调试模式
  • Warning 和 Notice 不一定是错误——可能是代码不规范但功能正常。Fatal error 才是真正的崩溃
  • 如果 debug.log 不存在或不生成，检查 wp-content 目录的写入权限
  • 调试模式会略微降低性能，不建议在生产环境长期开启

修改 NS 到 Hostinger

域名 L2

📖 What — 这是什么

NS（Name Server）记录指定由哪台 DNS 服务器来管理域名的 DNS 解析。修改 NS 到 Hostinger 意味着将域名的 DNS 管理权交给 Hostinger，之后所有 DNS 记录（A、CNAME、MX 等）都在 Hostinger 后台配置。

这是一种将域名绑定到主机的方式，替代手动添加 A 记录的方法。

🎯 Why — 为什么要做

当域名在第三方注册商（如 Namesilo、阿里云）购买，但主机使用 Hostinger 时，修改 NS 是最简单的绑定方式。Hostinger 会自动配置好所有需要的 DNS 记录。

与手动添加 A 记录相比，NS 方式让 Hostinger 全权管理 DNS，省去逐条配置的麻烦，适合新手。但代价是失去对 DNS 的精细控制。

🔧 How — 怎么做

  1. 登录 Hostinger hPanel，找到需要绑定的网站
  2. 在 域名 或 网站设置 中找到 Hostinger 的 NS 地址，通常是：
    • ns1.dns-parking.com
    • ns2.dns-parking.com

    （具体以 Hostinger 后台显示为准）

  3. 登录域名注册商后台（如 Namesilo）
  4. 找到域名管理 → Nameservers 设置
  5. 将默认 NS 替换为 Hostinger 提供的两条 NS 地址
  6. 保存更改
NS 修改需要 24-48 小时全球生效，期间网站可能间歇性无法访问。

⚠️ 常见问题与注意事项

  • NS 修改生效时间较长（最多 48 小时），比 A 记录慢得多
  • 修改 NS 后，原来在旧 DNS 上配置的所有记录（邮箱 MX、验证 TXT 等）都会失效，需要在 Hostinger 重新配置
  • 如果域名已经在 Cloudflare 管理 DNS，不建议改 NS 到 Hostinger，因为会失去 Cloudflare 的 CDN 和安全功能
  • 修改前建议截图保存当前所有 DNS 记录，方便迁移后重新配置
  • 可以用 nslookup -type=NS example.com 验证 NS 是否已更新

设置 WP_DEBUG_DISPLAY

wp-config L3

📖 What — 这是什么

WP_DEBUG_DISPLAY 控制是否将 PHP 错误/警告直接显示在网页上。它是 WordPress 调试三件套之一：

  • WP_DEBUG — 总开关，是否启用调试模式
  • WP_DEBUG_LOG — 是否将错误写入 wp-content/debug.log
  • WP_DEBUG_DISPLAY — 是否在页面上显示错误

🎯 Why — 为什么要做

开发调试时，推荐的组合是：开启 WP_DEBUG 和 WP_DEBUG_LOG，但关闭 WP_DEBUG_DISPLAY。这样错误信息会记录到日志文件，而不会暴露给前台访客。

在生产环境中显示 PHP 错误信息是严重的安全风险——它可能泄露文件路径、数据库结构、甚至敏感配置。

🔧 How — 怎么做

推荐的调试配置

# 开启调试模式
locwp wp wp-train -- config set WP_DEBUG true --raw

# 记录到日志文件
locwp wp wp-train -- config set WP_DEBUG_LOG true --raw

# 不在页面上显示
locwp wp wp-train -- config set WP_DEBUG_DISPLAY false --raw

对应的 wp-config.php 内容：

define( 'WP_DEBUG', true );
define( 'WP_DEBUG_LOG', true );
define( 'WP_DEBUG_DISPLAY', false );

查看当前设置

locwp wp wp-train -- config get WP_DEBUG_DISPLAY

查看日志文件

# 查看最新的错误日志
tail -50 wp-content/debug.log

⚠️ 常见问题与注意事项

  • 三个常量必须配合使用：只设 WP_DEBUG_DISPLAY 为 false 但没开 WP_DEBUG，没有任何效果
  • 生产环境务必关闭 WP_DEBUG_DISPLAY，防止信息泄露
  • Local 开发环境可以都设为 true 方便调试，但养成好习惯：日志优先于页面显示
  • debug.log 文件会持续增长，定期清理或设置 logrotate
  • 部分主机面板会覆盖 display_errors 的 PHP 设置，导致 WordPress 配置不生效

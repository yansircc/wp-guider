wp-content 目录结构

wp-config L2

📖 What — 这是什么

wp-content 是 WordPress 安装中最重要的自定义目录。WordPress 核心文件在更新时会被覆盖，但 wp-content 中的内容会被保留。

这个目录存放所有非核心的自定义内容：主题、插件、上传的媒体文件等。理解这个结构是 WordPress 文件管理的基础。

🎯 Why — 为什么要做

排障时需要知道文件在哪。插件冲突？去 plugins/ 看。上传的图片丢了？去 uploads/ 查。主题出问题？去 themes/ 改。

理解目录结构也是备份策略的基础——你需要知道哪些目录包含不可重建的数据（如 uploads），哪些可以通过重新安装恢复（如 plugins）。

🔧 How — 怎么做

wp-content 的标准子目录：

wp-content/
├── themes/        # 主题文件
├── plugins/       # 插件文件
├── uploads/       # 媒体上传（按年/月组织）
├── mu-plugins/    # 必须使用的插件（自动激活）
├── languages/     # 翻译文件
└── upgrade/       # 更新时的临时目录

用 WP-CLI 查看

# 查看已安装主题
locwp wp wp-train -- theme list

# 查看已安装插件
locwp wp wp-train -- plugin list

# 查看 wp-content 路径
locwp wp wp-train -- eval "echo WP_CONTENT_DIR;"

⚠️ 常见问题与注意事项

  • uploads/ 是唯一包含用户不可重建数据的目录，备份时必须包含
  • mu-plugins/ 目录中的插件无法从后台停用，只能通过文件操作管理
  • 不要手动修改 wp-content 以外的 WordPress 核心文件，更新时会被覆盖
  • 某些插件会在 wp-content 下创建自己的目录（如 cache/、w3tc-config/）
  • wp-config.php 中可以通过 WP_CONTENT_DIR 常量自定义此目录位置，但不建议修改

设置 WP_MEMORY_LIMIT

wp-config L3

📖 What — 这是什么

WP_MEMORY_LIMIT 是 WordPress 允许 PHP 使用的最大内存量。默认值是 40M（后台操作为 256M）。

当 WordPress 需要处理大量数据（如导入大文件、生成缩略图、运行复杂查询）时，可能会超出默认内存限制，导致"Allowed memory size exhausted"致命错误。

🎯 Why — 为什么要做

内存不足是 WordPress 常见的白屏/崩溃原因之一。特别是安装了多个插件或使用页面构建器（如 Elementor）时，内存消耗会显著增加。

合理设置内存限制是性能调优的基础步骤。设置过低导致崩溃，设置过高则浪费服务器资源。

🔧 How — 怎么做

查看当前内存限制

locwp wp wp-train -- config get WP_MEMORY_LIMIT

设置内存限制

# 设置为 256M（推荐值）
locwp wp wp-train -- config set WP_MEMORY_LIMIT 256M --raw

这会在 wp-config.php 中添加或更新：

define( 'WP_MEMORY_LIMIT', '256M' );

常用值参考

  • 64M — 小型站点，少量插件
  • 128M — 中等站点，10+ 个插件
  • 256M — 推荐值，适合大部分场景
  • 512M — 大型站点/页面构建器
WP_MEMORY_LIMIT 不能超过 PHP 本身的 memory_limit 设置。如果 PHP 限制为 128M，WordPress 设置 256M 也无效。

⚠️ 常见问题与注意事项

  • 值必须是字符串格式，带单位：'256M'，不要写成数字
  • 共享主机可能限制了最大 PHP 内存，无法通过 WordPress 配置突破
  • Local 环境默认 PHP 内存通常足够，一般不需要调整
  • 如果频繁遇到内存问题，应该排查是哪个插件/主题消耗过多内存，而不是一味增加限制
  • WP_MAX_MEMORY_LIMIT 控制后台管理区域的内存上限，默认 256M

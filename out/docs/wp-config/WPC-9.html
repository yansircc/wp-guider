SQL 查询基础

wp-config L4

📖 What — 这是什么

WordPress 的所有数据都存在 MySQL/MariaDB 数据库中。WP-CLI 的 db query 命令可以直接执行 SQL 语句，进行数据查询和修改。

WordPress 默认有 12 张核心表：posts、postmeta、options、users、usermeta、comments、commentmeta、terms、termmeta、term_taxonomy、term_relationships、links。

🎯 Why — 为什么要做

部分 WordPress 问题只能通过数据库层面解决。例如：重置管理员密码、批量修改文章状态、清理垃圾数据、排查性能瓶颈等。

WP-CLI 虽然封装了大部分常用操作，但有时直接写 SQL 更高效、更精确。

🔧 How — 怎么做

进入数据库交互模式

locwp wp wp-train -- db cli

常用查询示例

# 查看所有表
locwp wp wp-train -- db query "SHOW TABLES;"

# 查看 options 表结构
locwp wp wp-train -- db query "DESCRIBE wp_options;"

# 查询站点 URL
locwp wp wp-train -- db query "SELECT option_value FROM wp_options WHERE option_name = 'siteurl';"

# 查看最近 5 篇文章
locwp wp wp-train -- db query "SELECT ID, post_title, post_status FROM wp_posts WHERE post_type='post' ORDER BY ID DESC LIMIT 5;"

# 统计各状态的文章数
locwp wp wp-train -- db query "SELECT post_status, COUNT(*) as cnt FROM wp_posts WHERE post_type='post' GROUP BY post_status;"

数据修改（谨慎操作）

# 修改站点 URL（修复白屏时使用）
locwp wp wp-train -- db query "UPDATE wp_options SET option_value='http://correct-url.local' WHERE option_name='siteurl';"
UPDATE 和 DELETE 语句一旦执行无法撤销。务必先用 SELECT 确认要修改的数据，并在操作前创建数据库快照。

⚠️ 常见问题与注意事项

  • SQL 查询区分表前缀，确认用 wp_ 还是自定义前缀
  • WordPress 大量使用序列化数据（如 active_plugins），不能直接用 SQL 修改序列化字段
  • 执行 UPDATE/DELETE 前先 SELECT 确认影响范围
  • 操作前务必做数据库备份：locwp wp wp-train -- db export backup.sql
  • WP-CLI 的 search-replace 命令比手写 SQL 更安全，会正确处理序列化数据

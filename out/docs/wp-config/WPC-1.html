查看 DB_NAME

wp-config L2

📖 What — 这是什么

DB_NAME 是 wp-config.php 中定义的数据库名称常量。WordPress 的所有内容——文章、页面、用户、设置——都存储在这个数据库中。

wp-config.php 位于 WordPress 根目录，是整个站点的核心配置文件。其中数据库相关常量（DB_NAME、DB_USER、DB_PASSWORD、DB_HOST）决定了 WordPress 连接哪个数据库。

🎯 Why — 为什么要做

了解 DB_NAME 是数据库排障的起点。当遇到"Error establishing a database connection"错误时，首先要检查的就是数据库名是否正确。

在 Local (localWP) 环境中，每个站点的数据库名称由工具自动生成。知道如何查看 DB_NAME，才能正确连接数据库管理工具（如 Adminer/phpMyAdmin）进行数据操作。

🔧 How — 怎么做

方法一：用 WP-CLI 查看

locwp wp wp-train -- config get DB_NAME

输出就是当前站点使用的数据库名，例如 local。

方法二：直接查看文件

locwp wp wp-train -- eval "echo ABSPATH;"

找到 WordPress 根目录后，打开 wp-config.php，搜索 DB_NAME：

define( 'DB_NAME', 'local' );

其他数据库常量

  • DB_USER — 数据库用户名
  • DB_PASSWORD — 数据库密码
  • DB_HOST — 数据库主机（Local 通常是 localhost:/path/to/mysql.sock）

⚠️ 常见问题与注意事项

  • wp-config.php 包含数据库密码等敏感信息，绝不能公开分享或提交到版本控制
  • Local 环境中数据库名通常是 local，但其他环境可能不同
  • 如果 DB_NAME 指向的数据库不存在，WordPress 会报连接错误
  • 修改 wp-config.php 前务必备份，一个字符的错误就会导致白屏

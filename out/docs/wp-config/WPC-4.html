uploads 目录

wp-config L3

📖 What — 这是什么

wp-content/uploads/ 是 WordPress 存放所有通过媒体库上传的文件的目录。图片、文档、视频等媒体文件都保存在这里。

默认情况下，WordPress 按年和月份组织文件：uploads/2026/03/photo.jpg。这个行为可以在 设置 → 媒体 中关闭。

🎯 Why — 为什么要做

uploads 目录是整个 WordPress 中唯一包含用户不可重建数据的目录。主题和插件可以重新安装，数据库可以从 SQL 恢复，但用户上传的原始文件丢了就真的丢了。

理解 uploads 的结构对于备份策略、磁盘空间管理和迁移操作都至关重要。

🔧 How — 怎么做

查看 uploads 目录结构

# 查看上传目录路径
locwp wp wp-train -- eval "echo wp_upload_dir()['basedir'];"

# 查看目录大小
du -sh wp-content/uploads/

目录结构示例

uploads/
├── 2025/
│   ├── 01/
│   │   ├── hero-image.jpg
│   │   ├── hero-image-150x150.jpg    # 缩略图
│   │   └── hero-image-300x200.jpg    # 中等尺寸
│   └── 12/
├── 2026/
│   └── 03/
└── wpcf7_uploads/   # Contact Form 7 临时目录

WordPress 自动生成的图片尺寸

上传一张图片后，WordPress 会根据 设置 → 媒体 中的配置自动生成多个尺寸的副本：

  • 缩略图（Thumbnail）— 默认 150x150
  • 中等（Medium）— 默认最大 300x300
  • 大（Large）— 默认最大 1024x1024

⚠️ 常见问题与注意事项

  • 备份时 uploads 是最高优先级，其内容无法重建
  • 图片在数据库中只存储相对路径引用，实际文件在 uploads 目录中
  • 如果服务器磁盘空间不足，uploads 目录通常是最大的占用者
  • 迁移站点时，如果 uploads 路径或 URL 变化，需要用 WP-CLI 的 search-replace 更新数据库中的引用
  • 部分插件（如备份、表单插件）会在 uploads 下创建自己的子目录

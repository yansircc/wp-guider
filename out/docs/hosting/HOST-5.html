SSH 终端操作

主机 L4

📖 What — 这是什么

通过 SSH 连接到服务器后，可以在终端中使用 Linux 命令行操作文件、管理 WordPress（WP-CLI）、查看日志、排查问题等。这是 WordPress 高级管理的必备技能。

🎯 Why — 为什么要做

有些操作在 WordPress 后台或文件管理器中无法完成，或者效率极低。例如：批量修改文件权限、查找并替换数据库中的 URL、使用 WP-CLI 一条命令完成复杂操作。

当网站后台无法访问（白屏、500 错误）时，SSH 是唯一的排障入口。

🔧 How — 怎么做

常用命令速查

  • ls -la — 列出当前目录所有文件（含隐藏文件）
  • cd wp-content/plugins/ — 进入插件目录
  • cat wp-config.php — 查看 WordPress 配置文件
  • nano wp-config.php — 编辑配置文件（nano 编辑器）
  • find . -name "*.log" — 查找日志文件
  • du -sh * — 查看各目录大小
  • chmod 755 目录名 — 修改目录权限
  • chmod 644 文件名 — 修改文件权限

WP-CLI 常用命令

  • wp core version — 查看 WordPress 版本
  • wp plugin list — 列出所有插件及状态
  • wp plugin activate plugin-name — 激活插件
  • wp option get siteurl — 查看站点 URL
  • wp search-replace 'old-url' 'new-url' — 搜索替换

⚠️ 常见问题与注意事项

  • 操作前确认当前目录：pwd 查看，ls 确认文件
  • 编辑重要文件前先备份：cp wp-config.php wp-config.php.bak
  • 不要随意执行 rm -rf，删除操作不可逆
  • Hostinger 共享主机上 WP-CLI 已预装，可直接使用 wp 命令
  • 如果遇到 "Permission denied"，说明文件权限不足，需要检查文件所有者

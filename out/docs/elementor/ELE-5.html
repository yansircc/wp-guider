Grid Container 布局

Elementor 建站L3NEW

📖 What — 这是什么

Grid Container 是 Elementor 基于 CSS Grid 的布局模式，与默认的 Flexbox Container 互补。CSS Grid 擅长二维布局——同时控制行和列，适合创建卡片网格、图片画廊、定价表等规则网格结构。

创建 Container 时，在 Layout 设置中将 Container Type 从 Flex 切换为 Grid 即可启用。Grid 模式下你可以定义列数、行数和间距，子元素会自动填充到网格单元中。

🎯 Why — 为什么要做

Flexbox 擅长一维排列（单行或单列），但处理二维网格布局时需要多层嵌套。Grid 可以直接定义行列结构，一层 Container 就能实现复杂的网格布局，减少嵌套层级，代码更简洁。

典型场景：3×2 服务卡片网格、4 列图片画廊、不等宽的杂志式布局。

🔧 How — 怎么做

1. 创建 Grid Container

  1. 添加新 Container，点击编辑
  2. 在 Layout Tab 中，将 Container Type 改为 Grid
  3. 设置 Columns（列数）和 Rows（行数）
  4. 设置 Gaps（行间距和列间距）

2. 添加子元素

拖入 Widget 或子 Container 到 Grid 中，它们会自动按顺序填充网格单元。

3. 自定义网格位置

选中子元素 → Advanced Tab → 可以指定该元素占据的网格区域（Grid Column / Grid Row），实现跨行跨列布局。

4. 响应式调整

切换到平板/手机视图，可以为不同设备设置不同的列数。例如桌面 3 列、平板 2 列、手机 1 列。

Grid 和 Flex 可以混合使用——外层用 Grid 做网格，每个网格单元内部用 Flex Container 做内容排列。

⚠️ 常见问题与注意事项

  • Grid Container 需要 Elementor 3.12+，确保插件已更新到最新版
  • Grid 适合规则网格，不规则/流式布局仍然用 Flexbox 更合适
  • 如果子元素没有正确排列，检查是否在 Advanced Tab 中意外设置了 Grid 位置
  • Grid 布局在旧版浏览器（IE11）中不受支持，但现代浏览器完全兼容

Search Results 搜索结果页模板

Elementor 建站L3

📖 What — 这是什么

搜索结果页模板控制用户在站内搜索后看到的结果页面布局。WordPress 默认的搜索结果页样式通常很简陋，通过 Elementor Theme Builder 可以完全自定义搜索结果的展示方式。

搜索结果页本质上是一种特殊的 Archive 页面，所以在 Theme Builder 中归类在 Archive 下。

🎯 Why — 为什么要做

  • 用户体验：默认搜索结果页往往只有标题和摘要，缺少特色图和分类标签，不利于用户快速找到目标内容
  • 品牌统一：搜索页是高频访问页面，设计应与全站一致
  • "未找到"引导：可以在无结果时显示推荐内容或热门文章，减少用户流失

🔧 How — 怎么做

  1. 进入 Templates → Theme Builder
  2. 在 Archive 区域点击 "+" 新建模板
  3. 构建搜索结果页布局：
    • Archive Title：自动显示"搜索结果：关键词"
    • Search Form：在页面顶部放置搜索框，方便用户修改搜索词
    • Archive Posts：搜索结果文章列表，建议开启特色图和摘要显示
  4. 配置 Archive Posts 的 Skin 和分页方式
  5. Publish → Display Conditions → 选择 Archives → Search Results

"无搜索结果"处理

在搜索结果模板中，可以额外添加一个 Section，使用 CSS 条件控制仅在无结果时显示。或者直接在 Archive Posts Widget 的 Nothing Found Message 设置中自定义提示文字。

⚠️ 常见问题与注意事项

  • Display Conditions 必须选择 Search Results，而不是 All Archives — 否则会覆盖所有归档页
  • WordPress 默认搜索只搜索文章标题和内容，不搜索自定义字段。如需增强搜索，考虑 SearchWP 或 Relevanssi 插件
  • 搜索结果页的 URL 格式是 ?s=关键词，某些缓存插件默认不缓存带查询参数的页面，这是正常的
  • Preview Settings 中选择 "Search Results" 并输入一个搜索词来预览模板效果
  • 如果站点内容以产品为主（WooCommerce），可能需要单独处理产品搜索结果的展示

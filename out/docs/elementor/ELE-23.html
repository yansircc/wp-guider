手风琴、标签与目录 Widget

Elementor 建站L2

本文同时覆盖: ELE-23, ELE-24

📖 What — 这是什么

这三个 Widget 用于组织和导航大量内容：Accordion（手风琴） 和 Tabs（标签） 将内容分组折叠/切换显示；Toggle（切换）类似手风琴但可多项同时展开；Table of Contents（目录）自动根据页面标题层级生成导航目录。

🎯 Why — 为什么要做

  • 节省空间：FAQ、产品参数等长内容用手风琴折叠，页面更简洁
  • 改善体验：标签式切换让用户快速定位感兴趣的内容分类
  • 长文导航：TOC 帮助用户在长页面中快速跳转到目标章节
  • SEO 友好：FAQ 手风琴配合 Schema Markup，可以出现在 Google 搜索结果的 FAQ 富文本摘要中

🔧 How — 怎么做

Accordion 手风琴

  1. 拖入 Accordion Widget
  2. 每个 Item 设置标题和内容（支持富文本）
  3. 一次只能展开一个 Item（点击新的自动关闭前一个）
  4. Style Tab：自定义标题背景色、展开图标、间距
  5. 典型用途：FAQ 页面、产品规格、服务条款

Tabs 标签

  1. 拖入 Tabs Widget
  2. 每个 Tab 设置标签名和内容区
  3. 标签可以水平排列（默认）或垂直排列
  4. Pro 版的 Tabs 支持嵌套其他 Widget（包括其他 Elementor Widget）
  5. 典型用途：产品描述/参数/评价切换、不同方案对比

Toggle 切换

  • 与 Accordion 类似，但可以同时展开多个 Item
  • 适合"不确定用户想看哪些项"的场景

Table of Contents 目录

  1. 拖入 TOC Widget（Pro 功能）
  2. 自动扫描页面中的 H2-H6 标题生成目录
  3. 设置包含/排除哪些标题层级
  4. 可设为固定侧边栏（配合 Sticky 效果）
  5. Marker Type：数字、项目符号或无
  6. 支持折叠/展开最小化模式

⚠️ 常见问题与注意事项

  • 手风琴 vs 标签：内容条目多且每条较短用手风琴（如 FAQ），内容分类少但每块较长用标签（如产品详情）
  • 手机端标签：水平标签在手机上可能放不下，建议设置手机端自动切换为手风琴模式
  • TOC 锚点：TOC 基于标题的 ID 生成锚点链接。如果标题是中文，ID 可能不美观——可以在标题 Widget 的 Advanced → CSS ID 中手动设置英文 ID
  • SEO 注意：手风琴/标签内的内容 Google 可以正常抓取和索引，不需要担心隐藏内容的 SEO 问题

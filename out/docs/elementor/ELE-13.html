响应式设计

Elementor 建站L2-L3

本文同时覆盖: ELE-13, ELE-14, ELE-15

📖 What — 这是什么

响应式设计让你的页面在不同设备（桌面、平板、手机）上都有良好的展示效果。Elementor 提供三个预设断点，允许你针对每种设备分别调整布局、字号、间距和可见性。

🎯 Why — 为什么要做

  • 移动优先：超过 60% 的网页流量来自移动端，手机体验差直接损失客户
  • SEO 要求：Google 使用移动端体验作为排名因素（Mobile-First Indexing）
  • 用户体验：桌面上看起来完美的三栏布局，在手机上可能挤成一团

🔧 How — 怎么做

断点预览切换（L2）

  1. 在编辑器底部工具栏点击 响应式模式图标（电脑/平板/手机图标）
  2. 默认断点：Desktop（>1024px）、Tablet（768-1024px）、Mobile（<768px）
  3. 可在 Site Settings → Layout → Breakpoints 自定义断点值
  4. 切换到不同设备后，编辑器画布会模拟该设备宽度

按设备隐藏/显示元素（L3）

  1. 选中任意 Widget 或 Container
  2. 进入 Advanced Tab → Responsive
  3. 开启 Hide on Desktop / Hide on Tablet / Hide on Mobile
  4. 典型用法：桌面显示详细表格，手机端隐藏表格、改用卡片式列表

响应式排版（L3）

大多数数值属性都支持按设备设置不同值：

  1. 在属性旁看到 设备切换图标（🖥️📱）
  2. 点击切换到目标设备
  3. 设置该设备的专属值，例如：
    • 标题字号：Desktop 48px → Tablet 36px → Mobile 28px
    • Container padding：Desktop 60px → Mobile 20px
    • 列宽：Desktop 33.3% → Mobile 100%
  4. 值只在设定的设备上生效，不影响其他设备

响应式 Flexbox 技巧

  • Container 方向：Desktop 设为 Row（水平），Mobile 改为 Column（垂直堆叠）
  • Container Wrap：Desktop 设 No Wrap，Tablet 改为 Wrap 自动换行
  • 子元素顺序：通过 Advanced → Custom Ordering 调整 Mobile 下的排列顺序

⚠️ 常见问题与注意事项

  • 从桌面到手机：Elementor 采用"桌面优先"的继承方式——如果你只设了 Desktop 值，Tablet 和 Mobile 会继承它。只需要在需要调整的设备上设置不同值
  • 实机测试：编辑器的响应式预览是近似的，上线前务必用真实手机测试
  • 隐藏≠不加载：Hide on Mobile 只是 CSS display: none，内容仍然被加载。大量隐藏内容会影响手机性能
  • 字号规范：手机端正文不小于 16px，避免浏览器自动缩放导致布局错乱

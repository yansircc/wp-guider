Display Conditions 显示条件（不同页面不同模板）

Elementor 建站L4

📖 What — 这是什么

Display Conditions 是 Elementor Theme Builder 的条件显示系统。它决定每个模板（Header、Footer、Single、Archive 等）在哪些页面上生效。

通过组合 Include（包含）和 Exclude（排除）条件，你可以实现"首页用一个 Header，其他页面用另一个 Header"或"产品分类用网格归档模板，博客分类用列表归档模板"等复杂规则。

🎯 Why — 为什么要做

  • 差异化设计：不同类型页面的用户意图不同，需要不同的布局和信息架构
  • 精确控制：无需为每个页面单独编辑，通过规则自动匹配
  • 模板复用：创建少量模板，通过条件组合覆盖全站所有场景

🔧 How — 怎么做

设置 Display Conditions

  1. 在 Theme Builder 中编辑模板，点击 Publish（或已发布模板点击左下角的上箭头 → Display Conditions）
  2. 点击 Add Condition
  3. 选择条件层级：
    • Entire Site：全站生效
    • Archives：所有归档页 / 特定分类 / 特定标签 / 作者归档 / 日期归档
    • Singular：所有单页 / 特定 Post Type / 特定页面 / 特定文章 / 特定子页面
  4. 点击条件前的 Include/Exclude 切换按钮选择包含或排除
  5. 可以添加多条规则，它们之间是 OR（或）关系

条件优先级规则

当多个模板的条件都匹配同一个页面时，Elementor 按以下优先级决定使用哪个：

  1. 最具体的条件优先：指定了具体页面 ID 的条件 > 指定了分类的条件 > 全站条件
  2. 如果两个条件同样具体，后创建的模板优先

实际案例

  • Header A：Include → Entire Site（全站默认 Header）
  • Header B：Include → Singular → Front Page（首页专用 Header，更具体，优先级更高）
  • 结果：首页显示 Header B，其他所有页面显示 Header A

⚠️ 常见问题与注意事项

  • 在 Templates → Theme Builder 面板中，每个模板下方会显示其当前的 Display Conditions，方便全局查看
  • 如果某个页面没有匹配任何自定义模板，会回退到主题默认模板
  • Exclude 条件可以在 Include 基础上排除特定页面，如"全站生效，但排除首页"
  • 修改 Display Conditions 不需要重新编辑模板内容，直接在已发布模板上修改即可
  • 条件冲突时可以在 Theme Builder 面板查看哪些模板被标记了警告图标
  • 建议用命名规范标识模板用途，如"Header - 全站默认"、"Header - 落地页专用"

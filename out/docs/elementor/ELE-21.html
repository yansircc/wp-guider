Price Table 价格表 Widget

Elementor 建站L2NEW

📖 What — 这是什么

Price Table 是 Elementor Pro 的定价表 Widget，用于展示产品或服务的定价方案。每个 Price Table 包含方案名称、价格、功能列表和行动按钮，是 SaaS、服务类网站的标配组件。

通常以 2-4 列并排展示不同档位的方案（如基础版/专业版/企业版），用视觉差异引导用户选择推荐方案。

🎯 Why — 为什么要做

定价页面是转化漏斗的关键节点——用户在这里做最终的购买决策。清晰的价格对比可以减少决策摩擦，高亮推荐方案可以引导用户选择最有利的选项。

用 Price Table Widget 可以快速构建专业的定价对比布局，比手动拼凑 HTML 表格高效得多。

🔧 How — 怎么做

1. 创建价格表布局

  1. 创建一个 Row 方向的 Container
  2. 拖入 3 个 Price Table Widget（基础/专业/企业）
  3. 父 Container 设置 Gap 和 Align Items: Stretch

2. 配置每个 Price Table

Content Tab 分为多个区域：

  • Header：方案名称（如「专业版」）
  • Pricing：价格、货币符号、计费周期（/月、/年）
  • Features：功能列表，每项可设置图标（勾/叉）表示是否包含
  • Footer：CTA 按钮（如「立即购买」）
  • Ribbon：角标/标签（如「最受欢迎」「推荐」）

3. 高亮推荐方案

  • 给推荐方案开启 Ribbon（「推荐」「最受欢迎」）
  • 推荐方案使用品牌主色作为 Header 背景
  • 可以通过 Advanced Tab 设置推荐方案的上移效果（负 Margin Top）使其视觉突出
功能列表中用图标区分「包含」和「不包含」——绿色勾号 vs 灰色叉号，比纯文字更直观。

⚠️ 常见问题与注意事项

  • 价格表数量推荐 3 个（选择悖论：太多选项反而降低转化）
  • 功能列表保持对齐——各方案的功能项数量和顺序应一致，方便横向对比
  • 移动端 3 列会自动堆叠为纵向排列，确保堆叠后推荐方案在最前面
  • 价格数字要醒目——字号至少是正文的 2-3 倍

Popup 触发条件与显示规则

Elementor 建站L4

📖 What — 这是什么

Elementor Popup Builder 不仅能设计弹窗内容，还能精确控制何时弹出（触发条件）和在哪些页面弹出（显示规则）。触发条件包括页面加载延迟、滚动深度、退出意图、点击特定元素等；显示规则控制弹窗出现在全站还是特定页面。

🎯 Why — 为什么要做

  • 精准营销：在用户即将离开时弹出挽留窗口（Exit Intent），而非一进来就骚扰
  • 页面级控制：产品页弹询盘表单、博客页弹订阅框，不同页面不同弹窗
  • 频率控制：避免同一用户反复看到相同弹窗（设置 X 天内只显示一次）

🔧 How — 怎么做

设置触发条件（Triggers）

编辑 Popup → 左下角齿轮 → Advanced Rules Tab：

  • On Page Load：页面加载后 N 秒弹出（常用于欢迎/公告）
  • On Scroll：用户滚动到页面的 N% 位置时弹出
  • On Scroll to Element：滚动到指定 CSS 选择器的元素时弹出
  • On Click：用户点击指定元素（按钮/链接）时弹出
  • After Inactivity：用户无操作 N 秒后弹出
  • On Exit Intent（桌面端）：鼠标移向浏览器关闭按钮时弹出

设置显示规则（Conditions）

发布 Popup 时配置 Display Conditions：

  • Entire Site：全站显示
  • Specific Pages：仅在指定页面显示
  • Post Type / Category：仅在特定文章类型或分类下显示
  • Exclude：排除特定页面不显示

频率控制（Advanced Rules）

  • Show up to X times：每位用户最多看到 N 次
  • Don't show again for X days：用户关闭后 N 天内不再显示
  • Show on devices：限定桌面端/手机端/平板

⚠️ 常见问题与注意事项

  • Exit Intent 仅桌面端有效：手机端没有"鼠标移出"概念，移动端建议用滚动触发或延迟触发替代
  • 触发时机不宜过早：页面加载 0 秒就弹窗会极大伤害用户体验，建议至少延迟 5-10 秒
  • 频率控制很重要：不设频率限制 = 用户每次访问都弹窗 = 很快被讨厌
  • 多个 Popup 冲突：如果同一页面配置了多个 Popup，注意它们的触发条件不要重叠导致同时弹出

Theme Builder 主题构建器

Elementor 建站L3-L4

本文同时覆盖: ELE-36, ELE-37, ELE-38, ELE-39, ELE-40, ELE-41, ELE-42

📖 What — 这是什么

Theme Builder 是 Elementor Pro 的核心功能，让你用可视化方式设计网站的"结构层"——Header、Footer、文章模板、归档页、搜索结果页、404 页面等。不再受主题代码限制，完全掌控整站每个部分的设计。

🎯 Why — 为什么要做

  • 完全控制：不需要写 PHP 或修改主题文件，就能自定义网站每个结构部分
  • 一致性：所有页面共享同一套 Header/Footer，修改一处全站更新
  • 差异化：通过 Display Conditions，不同类型的页面可以使用不同模板
  • 脱离主题依赖：使用 Hello Elementor + Theme Builder，你的设计完全不依赖第三方主题

🔧 How — 怎么做

Header 页头（L3）

  1. 进入 Templates → Theme Builder → Header → 添加新模板
  2. 典型 Header 结构：
    • 一个水平 Container（Row 方向，Justify: space-between）
    • 左侧：Site Logo Widget
    • 中间：Nav Menu Widget（选择 WordPress 菜单）
    • 右侧：Search Widget + 按钮（CTA）
  3. 设置 Sticky Header：给 Header Container 添加 Sticky → Top
  4. 移动端汉堡菜单自动生成（Nav Menu Widget 内置响应式行为）

Footer 页脚（L3）

  1. Theme Builder → Footer → 添加新模板
  2. 典型 Footer 结构：
    • 主区域：3-4 列 Container，分别放置公司信息、快速链接、联系方式、社交媒体
    • 底部栏：版权信息 + 隐私政策链接
  3. 常用 Widget：Heading、Icon List（带图标列表）、Social Icons、Shortcode

Single Post 文章模板（L3）

  1. Theme Builder → Single Post → 添加新模板
  2. 使用 Post 专属 Widget：
    • Post Title（文章标题）
    • Featured Image（特色图）
    • Post Content（文章内容）
    • Post Info（日期/作者/分类）
    • Author Box（作者介绍框）
    • Post Navigation（上一篇/下一篇）
    • Post Comments（评论区）
    • Related Posts（相关文章）
  3. 这些 Widget 会自动拉取当前文章的数据（动态内容）

Archive 归档模板（L4）

  1. Theme Builder → Archive → 添加新模板
  2. 核心 Widget：Archive Title（自动显示分类名/标签名）+ Archive Posts（文章列表）
  3. Archive Posts 设置：
    • 布局：Classic（图+标题+摘要）/ Cards（卡片式）/ Full Content
    • 列数、每页文章数
    • 元数据显示（日期/作者/分类/评论数）
    • 分页样式

Display Conditions 显示条件（L4）

每个模板都需要设置"在哪里使用"：

  • Entire Site：全站应用（Header/Footer 通常选这个）
  • All Posts / All Pages：所有文章/所有页面
  • Posts in Category "X"：特定分类下的文章
  • Specific Post/Page：指定某个页面
  • 优先级规则：更具体的条件优先级更高。例如"所有文章用模板 A"+"产品分类用模板 B"，产品分类下的文章会使用模板 B

Search Results 搜索结果页（L3）

  • Theme Builder → Archive → 添加新模板，Display Condition 选 Search Results
  • 与 Archive 模板类似，使用 Archive Posts Widget 展示搜索结果
  • 添加 Search Form Widget 方便用户修改搜索词

404 页面（L3）

  • Theme Builder → Error 404 → 添加新模板
  • 典型内容：友好的提示信息、搜索框、热门页面链接、返回首页按钮
  • 设计应保持品牌一致性，让用户知道仍在你的网站上

⚠️ 常见问题与注意事项

  • 显示条件冲突：如果两个模板的 Display Condition 重叠，更具体的条件优先。在 Theme Builder 面板中可以看到哪些模板生效
  • 预览问题：编辑 Single Post 模板时，点击底部工具栏的"眼睛"图标选择一个真实文章来预览，否则看到的是默认占位内容
  • 主题兼容：使用 Theme Builder 需要主题支持。Hello Elementor 是最佳搭配；部分主题会与 Theme Builder 冲突
  • 全站计划：建议先画出站点结构图（哪些页面需要什么模板），再动手创建，避免返工
  • Nav Menu：在 WordPress 后台 外观 → 菜单 中先创建菜单，然后 Nav Menu Widget 才能选到
  • 404 测试：发布后访问一个不存在的 URL（如 /asdfgh）来验证 404 页面是否生效

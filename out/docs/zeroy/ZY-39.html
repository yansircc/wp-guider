排障：AI 生成报错（400/401/403 诊断）

ZeroY AI 建站 L3

📖 What — 这是什么

ZeroY 的 AI 内容生成功能依赖后端 API 调用。当 AI 生成失败时，浏览器控制台或页面上会出现 HTTP 错误码。三种最常见的错误：

  • 400 Bad Request — 请求格式错误，通常是参数缺失或无效
  • 401 Unauthorized — 认证失败，API 密钥无效或过期
  • 403 Forbidden — 有认证但没有权限，通常是配额耗尽或被限流

🎯 Why — 为什么要做

AI 生成是 ZeroY 建站的核心能力。如果 AI 不可用，用户要么手动写所有内容（低效），要么无法继续建站（阻塞）。快速诊断错误类型并修复是保证建站效率的关键。

🔧 How — 怎么做

第一步：获取错误信息

  1. 打开浏览器开发者工具（F12）→ Network 面板
  2. 触发 AI 生成操作
  3. 找到失败的请求（红色标记），查看：
    • Status Code：400 / 401 / 403
    • Response Body：具体的错误描述

400 Bad Request 排查

  • 原因：发送给 AI API 的参数格式不正确
  • 常见场景：prompt 为空、内容超出长度限制、请求体 JSON 格式错误
  • 解决：检查请求的 Request Body 内容，确认必要字段都已填写。如果是 prompt 为空，确保页面描述字段不为空后重试

401 Unauthorized 排查

  • 原因：API 密钥（API Key）无效、过期或未配置
  • 解决步骤：
    1. 进入 ZeroY 后台设置 → API 配置
    2. 检查 API 密钥是否已填写且完整（没有多余空格或截断）
    3. 到 API 提供商后台确认密钥状态（是否被吊销或过期）
    4. 重新生成密钥并粘贴到 ZeroY 配置中

403 Forbidden 排查

  • 原因：认证通过但被拒绝访问，通常是配额或限流问题
  • 常见场景：
    • API 调用配额用完（免费额度耗尽）
    • 短时间内请求过多被限流（Rate Limit）
    • 账户欠费或被暂停
  • 解决：登录 API 提供商后台查看用量和账户状态。如果是限流，等几分钟后重试。如果是配额，充值或升级计划
看错误码就够定位方向：400 查请求内容、401 查密钥、403 查配额。不用盲目排查。

⚠️ 常见问题与注意事项

  • API 密钥不要暴露：不要将 API 密钥发到群里、截图中或提交到 Git。泄露的密钥可能被滥用产生高额费用
  • 429 Too Many Requests：这是 403 的近亲，表示限流。等待 1-2 分钟后重试即可
  • 网络环境因素：如果在中国大陆直连海外 API，可能因网络不稳定导致超时。查看是否有代理或加速配置
  • debug.log 也有线索：ZeroY 可能将 API 调用错误写入 WordPress 的 debug.log，两个日志源对照查看效果更好
  • 多次重试无效时：记录完整的错误信息（状态码 + Response Body + 时间戳），联系 ZeroY 技术支持

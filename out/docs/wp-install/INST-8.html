已有站点更换域名

WordPress 安装 L4

📖 What — 这是什么

将已运行的 WordPress 站点从旧域名切换到新域名。涉及 DNS 配置、WordPress 内部 URL 替换、数据库内容批量更新以及旧域名 301 重定向等操作。

这不是简单地改个设置，因为 WordPress 会将域名硬编码在数据库的多处位置。

🎯 Why — 为什么要做

常见场景包括：品牌升级更换域名、从临时域名切换到正式域名、更换 TLD 后缀（如 .cn 换 .com）、从 HTTP 升级到 HTTPS 等。

如果只改了 DNS 而没有更新 WordPress 内部的 URL，站点会出现混合内容、图片加载失败、重定向循环等问题。

🔧 How — 怎么做

第一步：准备工作

  1. 完整备份数据库和文件（必须）
  2. 新域名的 DNS 已解析到主机 IP（A 记录或 NS 方式）
  3. 在主机后台绑定新域名

第二步：修改 WordPress 地址

方法一：通过 wp-config.php（推荐，最安全）

define('WP_HOME', 'https://新域名.com');
define('WP_SITEURL', 'https://新域名.com');

方法二：通过数据库直接修改

UPDATE wp_options SET option_value = 'https://新域名.com' WHERE option_name = 'siteurl';
UPDATE wp_options SET option_value = 'https://新域名.com' WHERE option_name = 'home';

第三步：批量替换数据库中的旧域名

WordPress 内容（文章、页面、小工具等）中包含大量旧域名引用。使用 WP-CLI 是最安全的方式：

wp search-replace 'https://旧域名.com' 'https://新域名.com' --all-tables

如果无法使用 WP-CLI，安装 Better Search Replace 插件，在后台执行搜索替换（注意勾选"Dry Run"先预览）。

第四步：设置 301 重定向

在旧域名的服务器或 DNS 配置 301 重定向，将所有流量导向新域名，保留 SEO 权重。

⚠️ 常见问题与注意事项

  • 操作前务必备份！URL 替换出错可能导致站点无法访问
  • 不要直接用 SQL 的 REPLACE() 函数，WordPress 有序列化数据，直接替换会破坏数据结构
  • 替换后如果登录后台时重定向循环，检查 wp-config.php 中是否有硬编码的旧域名
  • 更换域名后需要重新提交 Google Search Console 的站点地图
  • SSL 证书需要为新域名重新签发或更新
  • 如果使用了 CDN（如 Cloudflare），记得更新 CDN 中的域名配置
  • Elementor 等页面构建器会在自己的表中存储 URL，确保搜索替换覆盖了 --all-tables

绑定域名（A 记录方式）

WordPress 安装 L2

📖 What — 这是什么

通过 A 记录方式将自有域名绑定到 Hostinger 主机。与 NS 方式不同，A 记录方式不需要将整个域名的 DNS 管理权交给 Hostinger，只需要在当前 DNS 服务商（如 Cloudflare、阿里云）添加一条指向 Hostinger 服务器 IP 的 A 记录。

这种方式适合域名 DNS 已经在第三方平台管理、不想迁移 NS 的场景。

🎯 Why — 为什么要做

很多教练的域名已经在 Cloudflare 或阿里云管理 DNS，不希望改变 NS 指向。A 记录方式可以在保留现有 DNS 管理的同时，将域名指向 Hostinger 主机。

与 NS 方式相比，A 记录方式更灵活：可以只将主域名或某个子域名指向 Hostinger，其他子域名仍指向别处。

🔧 How — 怎么做

第一步：获取 Hostinger 服务器 IP

  1. 登录 Hostinger hPanel
  2. 进入 Hosting → Plan details
  3. 找到 IP Address（如 153.92.xx.xx），记录下来

第二步：在 DNS 服务商添加 A 记录

以 Cloudflare 为例：

  1. 登录 Cloudflare，选择对应域名 → DNS → Records
  2. 添加两条 A 记录：
    • Name：@，IPv4：Hostinger 的 IP 地址
    • Name：www，IPv4：同一个 IP 地址
  3. TTL 设为 Auto，Proxy 状态根据需要选择
  4. 保存记录

第三步：在 Hostinger 添加域名

  1. 回到 hPanel → Domains → Add Domain
  2. 选择 Use an Existing Domain
  3. 输入域名，选择 Point domain by A record
  4. 确认域名已正确绑定

⚠️ 常见问题与注意事项

  • DNS 记录生效需要时间，通常几分钟到几小时不等
  • 用 ping 你的域名 验证 A 记录是否已指向正确 IP
  • 如果同时使用 Cloudflare CDN（Proxy 模式），ping 结果是 Cloudflare IP 而非 Hostinger IP，这是正常的
  • 添加 A 记录前，确认没有同名的 CNAME 记录（会冲突）
  • 如果之前用 NS 方式绑定过，需要先将 NS 改回来再用 A 记录方式
  • SSL 证书需要在域名解析生效后才能正常签发，详见 SSL 配置文档

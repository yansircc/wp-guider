网站内容迁移 (手动数据库+文件)

WordPress 安装 L4

📖 What — 这是什么

手动迁移是指直接导出源站的数据库和 WordPress 文件，然后在目标主机上手动恢复。不依赖任何迁移插件，完全通过 phpMyAdmin（或命令行）和文件管理器（或 FTP/SSH）完成。

这是最底层的迁移方式，理解它有助于排查插件迁移失败时的问题。

🎯 Why — 为什么要做

插件迁移虽然方便，但在以下场景可能失败或不可用：大文件超过服务器上传限制、PHP 执行超时、插件不兼容、主机限制等。手动迁移没有这些限制，且能完全控制每个步骤。

作为教练，掌握手动迁移方法是排错和应急的必备技能。

🔧 How — 怎么做

第一步：导出源站数据库

  1. 登录源站主机的 phpMyAdmin
  2. 选择 WordPress 使用的数据库
  3. 点击 Export → Quick → Format: SQL
  4. 点击 Go，下载 .sql 文件

第二步：下载源站 WordPress 文件

  1. 通过文件管理器或 FTP 进入 WordPress 根目录
  2. 打包下载整个目录，重点是：
    • wp-content/（主题、插件、上传文件）
    • wp-config.php（数据库配置，需修改）

第三步：在目标主机恢复

  1. 在目标主机创建新的 MySQL 数据库和用户
  2. 通过 phpMyAdmin Import 导入 .sql 文件
  3. 将 WordPress 文件上传到目标主机的网站根目录
  4. 修改 wp-config.php 中的数据库信息： 
    define('DB_NAME', '新数据库名');
define('DB_USER', '新用户名');
define('DB_PASSWORD', '新密码');
define('DB_HOST', 'localhost');

第四步：更新 URL（如域名变更）

如果新站域名不同，需要在数据库中替换旧域名。在 phpMyAdmin 的 SQL 标签页执行：

UPDATE wp_options SET option_value = 'https://新域名' WHERE option_name = 'siteurl';
UPDATE wp_options SET option_value = 'https://新域名' WHERE option_name = 'home';

内容中的旧域名需要用搜索替换工具（如 Better Search Replace 插件）批量替换。

⚠️ 常见问题与注意事项

  • 导入数据库时如果报错文件过大，可以用命令行导入：mysql -u 用户名 -p 数据库名 < backup.sql
  • wp-config.php 的数据库表前缀（$table_prefix）必须与导入的数据库一致
  • 上传文件后检查文件权限：目录 755，文件 644
  • 迁移后如果出现白屏，检查 wp-config.php 中的数据库连接信息是否正确
  • 不要直接在数据库中用 SQL REPLACE 替换 URL，因为 WordPress 有序列化数据，建议用 WP-CLI 或专用插件
  • 迁移完成后清除所有缓存，包括浏览器缓存和服务器端缓存

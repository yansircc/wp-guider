删除用户并归属内容

用户管理 L2

📖 What — 这是什么

当需要移除一个用户时，WordPress 会询问如何处理该用户名下的所有内容（文章、页面、媒体等）。你可以选择将内容归属给另一个用户，或者删除所有内容。

这是一个需要谨慎操作的步骤，选错了可能导致内容丢失。

🎯 Why — 为什么要做

员工离职、外包结束、测试账号清理等场景都需要删除用户。正确处理内容归属能保证网站内容不丢失、SEO 不受影响。

如果直接选择"删除所有内容"，该用户发布的所有文章、上传的所有图片都会被永久删除，无法恢复。

🔧 How — 怎么做

  1. 登录 WordPress 后台
  2. 进入 用户 → 所有用户（Users → All Users）
  3. 将鼠标悬停在要删除的用户上，点击 删除
  4. 在确认页面，选择 "将所有内容归属给"（Attribute all content to），然后从下拉菜单选择接收内容的用户
  5. 点击 确认删除
务必选择"归属给"而非"删除所有内容"，除非你确定该用户没有任何有价值的内容。

⚠️ 常见问题与注意事项

  • 你不能删除自己正在使用的管理员账号
  • 站点必须至少保留一个管理员账号
  • 删除用户后，该用户的评论不会被删除，但评论会标记为匿名或保留原作者名
  • 建议在删除前先备份数据库，以防操作失误
  • 批量删除用户时同样会弹出内容归属选项，一定要仔细确认

themes/plugins 目录

wp-config L3

📖 What — 这是什么

wp-content/themes/ 存放所有已安装的主题，wp-content/plugins/ 存放所有已安装的插件。每个主题或插件是一个独立的子目录，包含 PHP、CSS、JS 等文件。

WordPress 通过扫描这些目录来发现可用的主题和插件。一个目录 = 一个主题/插件。

🎯 Why — 为什么要做

理解这两个目录的结构，是手动排障的必备技能。当后台无法访问时（如白屏），你需要直接操作文件系统来停用问题插件或切换主题。

例如：将 plugins/problem-plugin 重命名为 plugins/problem-plugin.bak，WordPress 就会自动停用该插件。

🔧 How — 怎么做

查看主题目录

# 列出所有主题
locwp wp wp-train -- theme list

# 查看当前激活的主题
locwp wp wp-train -- theme list --status=active

主题目录中的关键文件：

  • style.css — 包含主题元信息（名称、版本、作者）
  • functions.php — 主题功能定义，类似于主题专属的插件
  • templates/ — 页面模板文件

查看插件目录

# 列出所有插件
locwp wp wp-train -- plugin list

# 只看激活的
locwp wp wp-train -- plugin list --status=active

每个插件至少包含一个主 PHP 文件，文件头部有插件元信息注释。

文件级别停用插件

# 重命名即停用
mv plugins/problem-plugin plugins/problem-plugin.bak

# 恢复
mv plugins/problem-plugin.bak plugins/problem-plugin

⚠️ 常见问题与注意事项

  • 删除主题/插件目录会永久移除它们，包括所有自定义修改
  • WordPress 至少需要一个主题存在，否则前台会报错
  • 通过文件系统停用插件不会触发插件的 deactivation hook，部分插件可能留下残余数据
  • 不要直接修改第三方主题/插件的文件，更新时会被覆盖。应使用子主题或自定义插件

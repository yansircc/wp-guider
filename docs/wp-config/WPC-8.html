wp-cli 查看选项

wp-config L3

📖 What — 这是什么

WordPress 将大量运行时配置存储在数据库的 wp_options 表中。站点 URL、时区、固定链接结构、激活的插件列表等都在这里。

WP-CLI 的 option 命令可以直接读写这些选项，无需登录后台。这在排障和自动化操作中非常有用。

🎯 Why — 为什么要做

当后台无法访问（如白屏、登录循环）时，WP-CLI 是唯一能修改 WordPress 配置的途径。掌握 option 命令是 WordPress 运维的核心技能。

例如：修复错误的 siteurl 导致的无法登录问题，只能通过 WP-CLI 或直接操作数据库解决。

🔧 How — 怎么做

查看选项

# 查看站点 URL
locwp wp wp-train -- option get siteurl

# 查看首页 URL
locwp wp wp-train -- option get home

# 查看博客名称
locwp wp wp-train -- option get blogname

# 查看时区
locwp wp wp-train -- option get timezone_string

搜索选项

# 搜索包含 "mail" 的选项
locwp wp wp-train -- option list --search="*mail*"

# 以表格形式列出所有 autoload 的选项
locwp wp wp-train -- option list --autoload=yes --format=table

修改选项

# 修改站点标题
locwp wp wp-train -- option update blogname "我的站点"

# 修改时区
locwp wp wp-train -- option update timezone_string "Asia/Shanghai"

常用选项速查

  • siteurl — WordPress 安装地址
  • home — 站点访问地址
  • blogname — 站点标题
  • blogdescription — 站点副标题
  • active_plugins — 激活的插件列表（序列化数组）
  • template / stylesheet — 当前主题

⚠️ 常见问题与注意事项

  • 修改 siteurl 或 home 要特别小心，错误的值会导致站点完全无法访问
  • active_plugins 是序列化数组，不要手动编辑，应用 wp plugin activate/deactivate 命令
  • 部分选项修改后需要刷新固定链接才能生效
  • autoload 为 yes 的选项会在每次页面加载时被读取，过多的 autoload 选项会拖慢站点

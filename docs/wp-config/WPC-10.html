修复 siteurl

wp-config L4

📖 What — 这是什么

siteurl 和 home 是 WordPress 最关键的两个选项。siteurl 定义了 WordPress 核心文件的地址，home 定义了访客访问的地址。

当这两个值被错误修改时（如误改域名、多了斜杠、协议错误），整个站点会立即无法访问——后台进不去，前台也崩溃。这是 WordPress 最常见的"自己把自己锁在门外"的场景。

🎯 Why — 为什么要做

siteurl 错误是 WordPress 新手最常踩的坑之一。一旦改错，只能通过 WP-CLI、数据库直接操作或 wp-config.php 硬编码来修复。

掌握修复方法是 WordPress 排障的必备技能——因为这个问题迟早会遇到。

🔧 How — 怎么做

方法一：WP-CLI（推荐）

# 查看当前值
locwp wp wp-train -- option get siteurl
locwp wp wp-train -- option get home

# 修复为正确的 URL
locwp wp wp-train -- option update siteurl "http://wp-train.local"
locwp wp wp-train -- option update home "http://wp-train.local"

方法二：wp-config.php 硬编码

在 wp-config.php 中添加（会覆盖数据库中的值）：

define( 'WP_HOME', 'http://wp-train.local' );
define( 'WP_SITEURL', 'http://wp-train.local' );
这种方法会使后台"常规设置"中的地址字段变为灰色且不可编辑。修复后建议删除这两行，改回数据库管理。

方法三：SQL 直接修改

locwp wp wp-train -- db query "UPDATE wp_options SET option_value='http://wp-train.local' WHERE option_name='siteurl';"
locwp wp wp-train -- db query "UPDATE wp_options SET option_value='http://wp-train.local' WHERE option_name='home';"

修复后验证

# 确认值已正确
locwp wp wp-train -- option get siteurl
locwp wp wp-train -- option get home

# 清除缓存
locwp wp wp-train -- cache flush

⚠️ 常见问题与注意事项

  • siteurl 和 home 必须都修复，只改一个可能导致资源加载错误
  • URL 末尾不要加斜杠：用 http://wp-train.local，不要用 http://wp-train.local/
  • 如果站点使用了 SSL，URL 必须以 https:// 开头
  • 修复后如果 CSS/JS 仍然加载失败，可能需要用 search-replace 替换数据库中所有旧 URL 引用
  • 预防措施：在改动 URL 相关设置前，永远先记录当前值

查看 table_prefix

wp-config L3

📖 What — 这是什么

$table_prefix 是 wp-config.php 中定义的数据库表名前缀。默认值是 wp_。

WordPress 的所有数据库表都以这个前缀开头，例如 wp_posts、wp_options、wp_users。修改前缀可以让多个 WordPress 站点共用同一个数据库。

🎯 Why — 为什么要做

在进行数据库操作时，必须知道表前缀才能写出正确的 SQL 查询。例如查询所有文章，不是 SELECT * FROM posts，而是 SELECT * FROM wp_posts（假设前缀是 wp_）。

排障和数据恢复时，如果前缀不匹配，WordPress 会完全无法找到自己的数据。

🔧 How — 怎么做

方法一：WP-CLI 查看

locwp wp wp-train -- config get table_prefix

注意：table_prefix 不是 define() 常量，而是一个 PHP 变量，所以用 config get 即可。

方法二：查看 wp-config.php

$table_prefix = 'wp_';

验证数据库表

# 列出所有 WordPress 表
locwp wp wp-train -- db tables

输出类似：

wp_commentmeta
wp_comments
wp_options
wp_postmeta
wp_posts
wp_term_relationships
wp_term_taxonomy
wp_termmeta
wp_terms
wp_usermeta
wp_users

⚠️ 常见问题与注意事项

  • 安装后不要随意修改 table_prefix，否则 WordPress 找不到原有数据表
  • 如果导入 SQL 备份时表前缀不匹配，需要用文本替换修改 SQL 文件中的表名
  • table_prefix 只能包含数字、字母和下划线
  • 安全建议：使用非默认前缀（如 wpsite1_）可以增加 SQL 注入攻击的难度，但这不是主要防线

菜单显示位置

菜单与导航 L2

📖 What — 这是什么

WordPress 主题会注册若干「菜单显示位置」（Menu Location），常见的有：

  • Primary Menu / 主菜单 — 网站顶部的主导航
  • Footer Menu / 页脚菜单 — 网站底部的链接区
  • Mobile Menu / 移动端菜单 — 手机端的汉堡菜单

创建菜单后，必须将其分配到某个显示位置，前台才会渲染该菜单。一个菜单可以分配到多个位置，但一个位置只能绑定一个菜单。

🎯 Why — 为什么要做

创建菜单并添加好菜单项后，如果没有分配显示位置，前台不会显示任何导航。分配位置是让菜单"上线"的最后一步。

不同位置的菜单承担不同职责：顶部主导航用于核心页面入口，页脚菜单放隐私政策、条款等辅助链接。根据位置合理分配不同菜单。

🔧 How — 怎么做

方法一：在菜单编辑页分配

  1. 进入 外观 → 菜单，选择要分配的菜单
  2. 在页面底部找到 菜单设置 区域
  3. 在「显示位置」中勾选目标位置（如 Primary Menu）
  4. 点击 保存菜单

方法二：在「管理位置」统一分配

  1. 进入 外观 → 菜单，点击顶部 管理位置 选项卡
  2. 此页列出主题注册的所有位置
  3. 为每个位置在下拉菜单中选择对应的菜单
  4. 点击 保存更改
「管理位置」视图可以一眼看清所有位置的分配情况，适合统一调整。

⚠️ 常见问题与注意事项

  • 切换主题后菜单位置丢失：不同主题注册的位置名称不同，切换主题后需要重新分配。菜单本身和菜单项不会丢失
  • 看不到显示位置选项：确认你的主题确实注册了菜单位置。部分极简主题可能只注册了一个位置
  • 菜单已分配但前台不显示：检查主题是否在对应位置的模板中正确调用了 wp_nav_menu()。也可能是缓存问题，清除浏览器缓存再试
  • 同一菜单多个位置：一个菜单可以同时分配到主导航和移动端菜单位置，但可能因为样式不同导致显示效果差异

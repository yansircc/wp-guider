创建子页面

页面管理 L2

📖 What — 这是什么

WordPress 页面支持父子层级关系。一个「子页面」从属于某个「父页面」，在 URL 结构和后台管理中体现层级关系。例如，「服务介绍」页面下可以有「网站建设」「SEO 优化」等子页面。

子页面的 URL 会自动包含父页面的 slug，如 /services/web-design/，这种层级结构对 SEO 和用户导航都有帮助。

🎯 Why — 为什么要做

当网站内容较多时，扁平的页面结构会让后台管理变得混乱。通过父子层级，可以将相关页面组织在一起，逻辑清晰。

层级化的 URL 结构（如 /about/team/）让搜索引擎和用户都能更好地理解页面之间的关系，有利于 SEO 和用户体验。

🔧 How — 怎么做

  1. 确保父页面已经创建并发布
  2. 进入 页面 → 添加页面，创建新页面
  3. 在右侧面板中找到 页面属性（Page Attributes）区域
  4. 在「父级页面」下拉菜单中选择父页面
  5. 如果看不到页面属性面板，点击右上角 设置齿轮图标，确保「页面」选项卡已展开
  6. 编辑内容后点击 发布

发布后，在 页面 → 所有页面 列表中，子页面会以「— 子页面名」的缩进形式显示在父页面下方。

⚠️ 常见问题与注意事项

  • 找不到「页面属性」面板：在区块编辑器中，页面属性位于右侧的「页面」选项卡（不是「区块」选项卡），需要向下滚动才能看到
  • 层级不宜过深：建议最多 2-3 层，过深的层级会让 URL 过长，管理困难
  • 删除父页面的影响：如果删除父页面，子页面不会被删除，但会变成顶级页面，URL 结构随之改变
  • 排序：可在页面属性中设置「排序」数字来控制同级页面的显示顺序

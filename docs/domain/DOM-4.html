DNS A 记录解析

域名 L2

📖 What — 这是什么

A 记录（Address Record）是 DNS 中最基础的记录类型，作用是将域名指向一个 IPv4 地址。当用户在浏览器输入 example.com 时，DNS 通过 A 记录找到服务器的 IP 地址，然后浏览器连接到该服务器。

例如：example.com → 123.45.67.89 就是一条 A 记录。

🎯 Why — 为什么要做

将域名绑定到主机服务器是建站必经步骤。如果域名和主机不在同一个服务商（很常见），就需要手动添加 A 记录将域名指向主机的 IP 地址。

掌握 A 记录配置是教练的基本功，几乎每个建站项目都会用到。

🔧 How — 怎么做

以 Cloudflare DNS 为例（其他 DNS 服务商操作类似）：

  1. 登录 Cloudflare 控制台，选择对应的域名
  2. 进入 DNS → Records
  3. 点击 Add Record
  4. 填写：
    • Type：A
    • Name：@（代表根域名）或子域名（如 www）
    • IPv4 address：主机服务器的 IP 地址
    • TTL：Auto（或 300 秒）
    • Proxy status：Proxied（开启 CDN）或 DNS only
  5. 点击 Save

通常需要添加两条 A 记录：@（根域名）和 www，都指向同一个 IP。

⚠️ 常见问题与注意事项

  • DNS 生效需要时间（TTL 到期后），新记录通常几分钟生效，修改可能需要更长
  • 可以用 ping example.com 或在线工具（如 dnschecker.org）验证解析是否生效
  • 如果使用 Cloudflare Proxy（橙色云朵），ping 到的是 Cloudflare 的 IP，而非源站 IP，这是正常的
  • IP 地址要从主机服务商后台获取，不要猜测或使用过期 IP
  • 添加 A 记录前确认没有冲突的 CNAME 记录（同一名称不能同时有 A 和 CNAME）

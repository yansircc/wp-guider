DNS CNAME 记录

域名 L2

📖 What — 这是什么

CNAME 记录（Canonical Name Record）将一个域名别名指向另一个域名。与 A 记录直接指向 IP 不同，CNAME 是域名到域名的映射。

例如：www.example.com → example.com 就是一条 CNAME 记录。当访问 www.example.com 时，DNS 会先解析到 example.com，再通过 A 记录找到最终的 IP。

🎯 Why — 为什么要做

CNAME 最常见的用途是让 www 版本指向根域名。此外，很多第三方服务（如邮件服务、CDN、网站验证）要求你添加 CNAME 记录来完成配置。

CNAME 的好处是：当目标域名的 IP 变化时，你不需要修改 CNAME 记录，因为它指向的是域名而非固定 IP。

🔧 How — 怎么做

以 Cloudflare DNS 为例：

  1. 登录 Cloudflare 控制台，选择对应域名
  2. 进入 DNS → Records
  3. 点击 Add Record
  4. 填写：
    • Type：CNAME
    • Name：子域名（如 www、blog、shop）
    • Target：目标域名（如 example.com）
    • TTL：Auto
  5. 点击 Save

常见 CNAME 场景

  • www → example.com（www 跳转根域名）
  • mail → ghs.google.com（Google Workspace 邮箱）
  • _acme-challenge → 某值（SSL 证书验证）

⚠️ 常见问题与注意事项

  • 根域名（@）不能设置 CNAME（RFC 限制），只能用 A 记录。Cloudflare 支持根域名 CNAME Flattening 作为例外
  • 同一个名称不能同时有 CNAME 和 A 记录，CNAME 具有排他性
  • CNAME 会带来额外的 DNS 查询（先查 CNAME 再查目标的 A 记录），性能略有影响
  • 配置第三方服务的 CNAME 时，要完全按照服务商提供的值填写，不要自行修改

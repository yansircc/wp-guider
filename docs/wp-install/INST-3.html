绑定域名（NS 方式与 A 记录方式）

WordPress 安装 L2 本文同时覆盖: INST-3, INST-4

📖 What — 这是什么

将域名绑定到主机上有两种主要方式：修改 NS（Nameserver）和添加 A 记录。两种方式效果相同 — 都能让域名指向你的服务器 — 但实现路径和适用场景不同。

  • NS 方式：把域名的 DNS 管理权整体交给主机商，由主机商自动配置所有记录
  • A 记录方式：保持域名在原 DNS 服务（如 Cloudflare），只添加一条指向服务器 IP 的记录

🎯 Why — 为什么要做

WordPress 安装完成后，需要通过正式域名访问。绑定域名让用户通过 yourdomain.com 而非临时域名访问网站。

选择哪种方式取决于你的 DNS 管理需求：如果只有一个主机、不需要 Cloudflare CDN，用 NS 方式最简单；如果需要 Cloudflare 的 CDN/安全功能，或同一域名有多个子域名指向不同服务，用 A 记录方式。

🔧 How — 怎么做

方式一：NS 方式（简单，适合新手）

  1. 在 Hostinger hPanel 中找到 NS 地址（通常在域名设置页面）
  2. 登录域名注册商后台（Namesilo / 阿里云等）
  3. 进入域名管理 → Nameservers
  4. 将默认 NS 替换为 Hostinger 的 NS 地址
  5. 保存，等待 24-48 小时生效

方式二：A 记录方式（灵活，推荐进阶用户）

  1. 在 Hostinger hPanel 中找到服务器 IP 地址
  2. 登录 DNS 管理平台（如 Cloudflare）
  3. 添加两条 A 记录：
    • @ → 服务器 IP（根域名）
    • www → 服务器 IP（www 版本）
  4. 在 Hostinger hPanel → Domains 中添加该域名
  5. 等待 DNS 生效（通常几分钟到几小时）

对比表

  • NS 方式：设置简单 / 生效慢（24-48h）/ DNS 由主机商管理 / 不兼容 Cloudflare CDN
  • A 记录方式：设置稍复杂 / 生效快（分钟级）/ 保留原 DNS 控制权 / 兼容 Cloudflare

⚠️ 常见问题与注意事项

  • NS 方式会覆盖原有的所有 DNS 记录（邮箱 MX、验证 TXT 等），切换前务必备份
  • A 记录方式需要在 Hostinger 后台也添加域名，否则服务器不会响应该域名的请求
  • 绑定域名后，WordPress 后台的 设置 → 常规 中的 URL 也需要更新为正式域名
  • 如果使用 Cloudflare + A 记录，建议同时开启 SSL（Full Strict 模式）
  • 域名绑定后如果仍无法访问，用 ping yourdomain.com 检查 DNS 解析是否正确

网站迁移与域名更换

WordPress 安装 L4 本文同时覆盖: INST-6, INST-7, INST-8

📖 What — 这是什么

网站迁移是将 WordPress 站点从一个服务器搬到另一个服务器的过程。域名更换则是在同一服务器上将站点的访问地址从旧域名改为新域名。两者经常同时发生，技术核心是相同的：文件转移 + 数据库转移 + URL 替换。

迁移方式分为插件方式（简单）和手动方式（可控）。

🎯 Why — 为什么要做

  • 更换主机：服务器性能不足、价格调整、服务商跑路等
  • 更换域名：品牌升级、域名过期被抢注、从临时域名切换到正式域名
  • 开发到上线：从本地/暂存环境迁移到生产服务器

这是建站服务商的核心技能之一，几乎每个项目都会涉及。

🔧 How — 怎么做

方式一：插件迁移（推荐新手）

使用 All-in-One WP Migration 插件：

  1. 在源站安装并激活 All-in-One WP Migration 插件
  2. 进入 All-in-One WP Migration → 导出
  3. 选择 导出到 → 文件，下载 .wpress 备份文件
  4. 在目标站安装全新的 WordPress 并激活同一插件
  5. 进入 All-in-One WP Migration → 导入
  6. 上传 .wpress 文件，等待导入完成
  7. 导入后需要重新设置固定链接（设置 → 固定链接 → 直接保存）
如果更换了域名，插件会自动执行搜索替换，将旧域名替换为新域名。

方式二：手动迁移（数据库 + 文件）

  1. 导出数据库：通过 phpMyAdmin 或 wp db export 导出 SQL 文件
  2. 打包文件：将整个 WordPress 目录打包下载（或通过 SSH tar -czf）
  3. 上传文件：将文件上传到目标服务器并解压
  4. 创建数据库：在目标服务器创建新的 MySQL 数据库和用户
  5. 修改 wp-config.php：更新数据库名、用户名、密码、主机
  6. 导入数据库：通过 phpMyAdmin 或 wp db import 导入 SQL 文件
  7. 搜索替换 URL：使用 WP-CLI 替换域名 
    wp search-replace 'https://old-domain.com' 'https://new-domain.com' --all-tables
  8. 更新固定链接：后台设置 → 固定链接 → 保存（刷新 .htaccess）

方式三：已有站点仅更换域名

  1. 将新域名的 DNS 指向同一服务器
  2. 在主机控制面板中添加新域名
  3. 使用 WP-CLI 搜索替换： 
    wp search-replace 'old-domain.com' 'new-domain.com' --all-tables
wp cache flush
  4. 在旧域名配置 301 重定向到新域名（保持 SEO 权重）
  5. 更新 Google Search Console 和 GA 中的域名设置

⚠️ 常见问题与注意事项

  • 迁移前必须备份：源站和目标站都要备份，出问题可以回滚
  • All-in-One WP Migration 免费版有上传大小限制（通常 256MB），超过需要购买插件或手动迁移
  • WordPress 数据库中的 URL 是序列化存储的（serialized），不能用简单的文本替换，必须用 WP-CLI 的 search-replace 或专用工具
  • 迁移后如果图片显示异常，检查 wp-content/uploads/ 目录是否完整转移
  • 更换域名后，旧域名至少保留 6-12 个月做 301 重定向，否则 SEO 排名会断崖式下跌
  • 迁移后登录密码是源站的密码，不是目标站安装时设置的密码
  • Elementor 有自己的 URL 替换工具（Elementor → 工具 → 替换 URL），迁移后需要额外执行

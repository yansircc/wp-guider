清除 Hostinger 缓存（页面缓存 + CDN + 对象缓存）

ZeroY AI 建站L2NEW

📖 What — 这是什么

Hostinger 主机有多层缓存：LiteSpeed 页面缓存、CDN 缓存、对象缓存（Object Cache）。当你修改了页面内容但前台看不到变化时，通常是缓存在作怪。你需要逐层清除缓存才能看到最新内容。

🎯 Why — 为什么要做

缓存是最常见的「改了但看不到效果」的原因。尤其在 ZeroY 编辑器中保存修改后，如果前台没有更新，首先要做的就是清缓存。不了解缓存清除流程会浪费大量时间在排查上。

🔧 How — 怎么做

1. 清除页面缓存（LiteSpeed Cache）

  1. WordPress 后台 → LiteSpeed Cache → 工具箱（Toolbox）
  2. 点击 Purge All（清除全部缓存）

2. 清除 CDN 缓存

  1. 登录 Hostinger 控制面板（hPanel）
  2. 找到 性能 → CDN 或 缓存管理
  3. 点击清除 CDN 缓存

3. 关闭/清除对象缓存

  1. Hostinger hPanel → 高级 → 对象缓存
  2. 如果 LiteSpeed 对象缓存导致问题，关闭它
  3. 或在 LiteSpeed Cache 插件中 → 缓存 → 对象缓存 → 关闭
LiteSpeed 对象缓存有时会和 ZeroY 冲突，如果遇到编辑器异常，优先尝试关闭对象缓存。

4. 浏览器强制刷新

清完服务端缓存后，在浏览器中按 Ctrl+Shift+R（Mac 为 Cmd+Shift+R）强制刷新页面。

⚠️ 常见问题与注意事项

  • 清缓存的顺序：WordPress 插件缓存 → Hostinger CDN → 浏览器缓存，逐层清除
  • 清缓存不会丢失数据：缓存只是页面快照，清除后系统会重新生成
  • 频繁开发时可关闭缓存：开发调试阶段可以暂时关闭 LiteSpeed Cache 插件，上线后再开启
  • 用无痕窗口验证：无痕窗口没有浏览器缓存，是最快的验证方式

HTMX 基础：Load More 加载更多

ZeroY AI 建站L3

📖 What — 这是什么

HTMX 是 ZeroY 集成的前端交互技术，无需编写 JavaScript 就能实现动态页面效果。"Load More"（加载更多）是最基础的 HTMX 功能：用户点击按钮后，页面自动请求更多内容并追加显示，无需刷新整个页面。

🎯 Why — 为什么要做

当文章或产品列表较长时，一次性加载所有内容会导致页面慢且臃肿。Load More 按钮让用户按需加载，既保持首屏加载速度，又提供流畅的浏览体验。比传统分页更符合现代用户习惯。

🔧 How — 怎么做

  1. 在 ZeroY 编辑器中创建文章/产品列表区块
  2. 为列表区块配置 HTMX 属性，指定数据源端点： 
    /wp-json/zeroy/v1/htmx/execute/current?block_id=你的区块ID
  3. 在列表下方添加 "加载更多" 按钮
  4. 为按钮配置 HTMX 点击事件：
    • 触发方式：click
    • 请求目标：列表容器
    • 追加方式：在现有内容后面追加（append/afterend）
  5. 设置分页参数（每次加载多少条）
  6. 保存并测试

⚠️ 常见问题与注意事项

  • 端点格式：HTMX 请求的端点必须是 /wp-json/zeroy/v1/htmx/execute/current?block_id=...，不要自行拼接其他路径
  • 固定链接：REST API 依赖"文章名"固定链接格式，否则端点不可用
  • 没有更多数据时：当所有内容加载完毕，按钮应自动隐藏或变为"没有更多了"
  • 缓存干扰：页面缓存可能导致 HTMX 请求返回过期数据，必要时排除 REST API 路径的缓存

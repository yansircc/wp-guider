排障：表单提交后不跳转/不发邮件

ZeroY AI 建站 L3

📖 What — 这是什么

ZeroY 表单提交依赖 WordPress REST API 和邮件发送系统。常见的故障表现：

  • 提交后无反应/不跳转：表单提交按钮点击后页面没有任何变化
  • 提交成功但不发邮件：数据到了后台但通知邮件收不到

这两类问题的根因通常是三个方面之一：固定链接设置、SMTP 邮件配置、REST API 路径。

🎯 Why — 为什么要做

表单是网站的核心转化入口——如果访客填完表单却提交失败，或提交成功但你收不到通知，就等于丢失潜在客户。表单故障是最紧急的线上问题之一。

🔧 How — 怎么做

排查一：固定链接（最常见原因）

ZeroY 表单通过 /wp-json/zeroy/v1/submissions 提交。如果固定链接设为「朴素」模式，REST API 路径不生效。

  1. 进入 设置 → 固定链接
  2. 确认不是「朴素」（Plain）模式，建议选「文章名」
  3. 即使当前已经是正确设置，也点击一次 保存更改（会刷新 .htaccess / rewrite 规则）

验证 REST API 是否可用：

# 浏览器直接访问，应返回 JSON
https://your-site.com/wp-json/zeroy/v1/submissions

# 或用命令行
curl -s https://your-site.com/wp-json/ | head -20

排查二：SMTP 邮件配置

WordPress 默认用 PHP mail() 函数发邮件，但大多数服务器/主机屏蔽了这个函数。需要配置 SMTP。

  1. 安装并激活 WP Mail SMTP 插件
  2. 进入 WP Mail SMTP → 设置
  3. 配置 SMTP 服务器信息（发件人邮箱、SMTP 主机、端口、认证方式）
  4. 使用 WP Mail SMTP → 工具 → 邮件测试 发送测试邮件
  5. 确认测试邮件已收到

排查三：REST API 路径问题

如果 REST API 被安全插件阻止，或 Nginx/Apache 配置不转发 /wp-json/ 路径：

  1. 检查浏览器控制台（F12 → Network），看表单提交请求的 HTTP 状态码
  2. 404 → 固定链接或 rewrite 规则问题
  3. 403 → 安全插件或服务器防火墙阻止了 REST API
  4. 500 → 服务端错误，查看 debug.log
先排查固定链接（1 分钟搞定），再查 SMTP（5 分钟），最后才查 REST API 路径（需要看日志）。按简单到复杂的顺序排查。

⚠️ 常见问题与注意事项

  • LocalWP 环境不发邮件是正常的：本地开发环境默认没有邮件服务，邮件会进入 LocalWP 内置的 MailHog。在 LocalWP 界面点击「Open MailHog」查看
  • 检查垃圾邮件箱：SMTP 配置正确但收不到邮件时，先检查收件箱的垃圾邮件文件夹
  • 表单提交数据是否丢失：即使邮件没发出，提交的数据通常已保存在数据库中。进入 ZeroY 后台查看提交记录
  • 浏览器控制台是关键工具：表单提交问题 90% 可以通过 F12 → Network 面板看到原因（HTTP 状态码 + 响应内容）

Google Ads 转化代码 + 全局代码配置

ZeroY AI 建站L3

本文同时覆盖: ZY-25, ZY-26

📖 What — 这是什么

Google Ads 转化追踪需要在网站中放置两段代码：全局代码（Global Site Tag）放在每个页面，用于追踪访客行为；事件代码（Event Snippet）放在感谢页，用于记录具体的转化动作（如表单提交）。

🎯 Why — 为什么要做

没有转化追踪，你投放 Google Ads 就是"盲投"——不知道哪些关键词、广告组、搜索词带来了真实的询盘。正确配置转化追踪后，Google Ads 后台能看到每个广告的转化数据，为优化投放提供依据。

🔧 How — 怎么做

第一步：获取代码

  1. 登录 Google Ads 后台
  2. 进入 工具与设置 → 转化 → 新建转化操作 → 网站
  3. 按提示设置转化名称、类别（如"询盘"）、价值等
  4. 获取两段代码：全局代码和事件代码

第二步：放置全局代码（通过 SEO 插件）

  1. 进入 WordPress 后台 → SEOPress → Analytics（或其他 SEO 插件的对应设置）
  2. 找到 Custom Tracking / Header Scripts 区域
  3. 将 Google 全局代码（<script> 标签）粘贴到 Header 脚本区域
  4. 保存

第三步：放置事件代码（在感谢页）

  1. 用 ZeroY 编辑器打开感谢页（Thank You Page）
  2. 在页面底部添加一个自定义 HTML 区块
  3. 粘贴 Google Ads 事件代码（Event Snippet）
  4. 保存页面

第四步：验证

  1. 使用 Google Tag Assistant 浏览器扩展检查代码是否正常加载
  2. 提交一次测试表单，跳转到感谢页后检查 Google Ads 后台是否记录到转化

⚠️ 常见问题与注意事项

  • 全局代码放 Header，事件代码放感谢页：两段代码用途不同，不要搞混位置
  • 不要重复放置：如果 SEO 插件已经放了全局代码，不要在主题设置中再放一次，否则会重复计数
  • 转化延迟：Google Ads 后台的转化数据可能有 24-72 小时的延迟，不要着急
  • 感谢页 noindex：感谢页应设为 noindex，但不影响转化追踪代码的执行
  • SEOPress 路径：SEOPress → Analytics → Custom Tracking，其他 SEO 插件类似功能的位置可能不同

Dynamic Tags 基础（站点标题/文章标题/特色图）

Elementor 建站L3

📖 What — 这是什么

Dynamic Tags 是 Elementor Pro 的动态内容功能。它允许 Widget 的内容不再是固定文字或图片，而是从 WordPress 数据库动态拉取 — 站点标题、文章标题、特色图、作者名、发布日期、自定义字段等。

Dynamic Tags 是 Theme Builder 的基础 — 正是因为有了 Dynamic Tags，一个 Single Post 模板才能自动适配所有文章。

🎯 Why — 为什么要做

  • 自动化：无需手动为每篇文章或页面重复输入信息
  • 一处修改全站生效：修改站点标题后，所有使用了 Dynamic Tag 的地方自动更新
  • 模板前提：没有 Dynamic Tags 就无法创建真正的动态模板

🔧 How — 怎么做

插入 Dynamic Tag

  1. 选中一个 Widget（如 Heading）
  2. 在内容输入框旁找到 Dynamic Tags 图标（数据库/堆栈图标）
  3. 点击后弹出 Dynamic Tags 选择面板
  4. 选择数据源，常用的有：
    • Site：Site Title（站点标题）、Site Tagline（副标题）、Site Logo
    • Post：Post Title（文章标题）、Post Excerpt（摘要）、Post Date、Featured Image
    • Author：Author Name、Author Bio、Author Profile Picture
    • Archive：Archive Title、Archive Description

Dynamic Tags 可用于哪些属性

不只是内容，Dynamic Tags 还可以用于：

  • 链接：按钮的 URL 动态指向文章链接
  • 图片：Image Widget 动态显示特色图或 ACF 图片字段
  • 背景图：Section 背景动态使用特色图
  • 颜色 / 数字：部分属性支持动态颜色或数值

Fallback 设置

点击 Dynamic Tag 后可以设置 Fallback（回退值）。当动态数据为空时（如文章没有特色图），显示 Fallback 指定的默认内容。

⚠️ 常见问题与注意事项

  • Dynamic Tags 是 Elementor Pro 功能，免费版没有
  • 在 Theme Builder 模板中，Dynamic Tags 会根据当前页面上下文自动取值。在普通页面编辑器中，Dynamic Tags 取的是当前页面的数据
  • 设置 Fallback 是好习惯 — 避免数据缺失时页面出现空白区域
  • 不是所有 Widget 属性都支持 Dynamic Tags，支持的属性旁边会显示 Dynamic 图标
  • Dynamic Tags 的值在编辑器中可能显示为占位符，需要在前端预览或已发布页面中查看实际效果

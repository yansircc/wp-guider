嵌套 Container（多层布局）

Elementor 建站L2

📖 What — 这是什么

嵌套 Container 是指在一个 Container（容器）内部再放置子 Container，形成多层布局结构。这是实现复杂页面设计的核心技术——比如一个外层 Container 控制整体宽度和背景，内部多个子 Container 分别控制左右分栏。

Elementor 的 Container 基于 CSS Flexbox，父容器可以控制子元素的排列方向（横向/纵向）、对齐方式和间距，每个子 Container 又可以有自己独立的 Flexbox 设置。

🎯 Why — 为什么要做

单层 Container 只能做简单的线性布局。现实中的页面设计几乎都需要嵌套——Hero 区域的文字+按钮组合、卡片网格中每张卡片的内部结构、页脚的多列布局等。

理解嵌套逻辑后，你可以用 Container 构建任意复杂的布局，而不依赖固定的 Section/Column 模式。

🔧 How — 怎么做

1. 创建外层 Container

点击画布中的 「+」，选择一个全宽 Container。设置方向为 Row（横向排列子元素）。

2. 添加子 Container

  1. 从左侧 Widget 面板拖入一个 Container Widget 到外层 Container 内
  2. 重复拖入第二个 Container，形成左右两栏
  3. 每个子 Container 可以独立设置宽度（百分比或固定值）

3. 设置子 Container 的布局

  • 子 Container 默认方向为 Column（纵向堆叠），可在 Layout Tab 中修改
  • 设置 Width：如左栏 60%，右栏 40%
  • 设置 Gap（间距）控制子元素之间的距离

4. 在子 Container 中放置 Widget

在左侧子 Container 中拖入 Heading + Text Editor，在右侧放入 Image Widget，即可实现经典的图文分栏布局。

使用 Navigator（左下角图标）可以清楚看到容器嵌套层级，拖拽调整结构比在画布上操作更精确。

⚠️ 常见问题与注意事项

  • 嵌套层级不宜过深——一般 2-3 层足够，过深会增加 DOM 复杂度影响性能
  • 如果子 Container 没有按预期排列，检查父 Container 的 Direction 设置（Row vs Column）
  • 子 Container 的宽度总和超过 100% 时会换行——确保百分比加起来不超过 100%（加上 Gap 的影响）
  • 想要等高列（每列高度一致），在父 Container 的 Items 属性中设置 Align Items: Stretch

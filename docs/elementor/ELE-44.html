ACF + Dynamic Tags（自定义字段动态显示）

Elementor 建站L4

📖 What — 这是什么

ACF（Advanced Custom Fields）为 WordPress 添加自定义字段，Elementor 的 Dynamic Tags 可以直接读取这些字段并显示在前端。两者结合，可以实现"后台填数据、前端自动展示"的动态内容系统。

例如：给产品文章添加 ACF 字段"价格"、"规格"、"交付时间"，然后在 Elementor 模板中用 Dynamic Tags 调用这些字段，每篇产品文章自动显示各自的信息。

🎯 Why — 为什么要做

  • 结构化数据：将信息拆分为独立字段，而非全部塞在文章正文里，方便管理和展示
  • 前后端分离：内容编辑者只需在后台填写字段值，无需关心前端样式
  • 灵活展示：同一个 ACF 字段可以在不同位置、以不同样式展示
  • 实战必备：企业站、产品站、房产站等几乎所有商业项目都需要自定义字段

🔧 How — 怎么做

前提条件

  • 已安装 ACF（免费版或 Pro）并创建了字段组
  • 已安装 Elementor Pro

在 Elementor 中调用 ACF 字段

  1. 在 Theme Builder 中编辑 Single Post 模板（或任何需要显示 ACF 数据的模板）
  2. 添加一个 Widget（如 Heading 显示文本字段，Image 显示图片字段）
  3. 点击 Dynamic Tags 图标
  4. 在 Dynamic Tags 面板中找到 ACF 分组
  5. 选择对应的 ACF 字段

不同字段类型的对应关系

  • Text / Number / Email / URL → Heading、Text Editor、Button（链接）等文本类 Widget
  • Image → Image Widget 或 Section 背景图
  • True/False → 用于 Conditions 控制元素是否显示（需配合插件如 Dynamic Visibility）
  • Repeater → 需要 Elementor Pro 的 Loop Builder 或第三方插件（如 Dynamic Content for Elementor）
  • Gallery → Gallery Widget 或 Image Carousel

⚠️ 常见问题与注意事项

  • 如果 Dynamic Tags 面板中看不到 ACF 字段，确认 ACF 字段组的"位置规则"包含了当前正在编辑的 Post Type
  • ACF Repeater 字段不能直接用 Dynamic Tags 读取子字段，需要额外处理
  • 在编辑器中 ACF 字段可能显示为空（因为模板没有实际文章上下文），在 Preview Settings 中指定一篇有数据的文章来预览
  • ACF 字段输出的内容是纯数据，样式完全由 Elementor Widget 控制 — 这是正确的前后端分离
  • 给 ACF 字段设置合理的默认值，避免前端出现空白区域

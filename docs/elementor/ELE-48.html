WooCommerce 商店归档 + 购物车 / 结账页

Elementor 建站L4NEW

📖 What — 这是什么

Elementor Pro + WooCommerce 结合后，可以用 Theme Builder 完全自定义商店的三大核心页面：商品归档页（Shop / Category 页）、购物车页（Cart）、结账页（Checkout）。你可以用 Elementor 的可视化方式替代 WooCommerce 默认模板，获得更自由的设计控制。

🎯 Why — 为什么要做

  • 品牌统一：默认 WooCommerce 模板风格固定且与品牌脱节，自定义后整站视觉一致
  • 转化优化：优化购物车和结账页的布局可显著降低弃购率
  • 灵活展示：归档页可以自定义筛选器、排序方式、卡片样式，突出产品卖点

🔧 How — 怎么做

商品归档页（Shop / Category）

  1. WordPress 后台 → 模板 → Theme Builder → Product Archive
  2. 新建模板 → 选择 Elementor 编辑
  3. 使用 WooCommerce 专用 Widget：
    • Archive Products：产品网格（控制列数、卡片样式、每页数量）
    • Archive Title：当前分类名称
    • Archive Description：分类描述
    • Products Filter：价格/属性/分类筛选（如果可用）
  4. 配置 Display Conditions：应用于 Shop 页面 + 所有产品分类页

购物车页（Cart）

  1. Theme Builder → 新建 Cart 类型模板
  2. 使用 Cart Widget（Elementor 封装的购物车表格）
  3. 自定义按钮样式、配色、空购物车提示文案
  4. 添加交叉销售（Cross-sells）Widget 推荐相关产品

结账页（Checkout）

  1. Theme Builder → 新建 Checkout 类型模板
  2. 使用 Checkout Widget 布局表单字段
  3. 可以用两列布局：左侧账单信息，右侧订单摘要
  4. 自定义"下单"按钮样式和位置

⚠️ 常见问题与注意事项

  • 需要 Elementor Pro：WooCommerce Theme Builder 是 Pro 功能
  • 支付网关兼容性：自定义结账页后要测试所有启用的支付方式是否正常工作
  • 不要删除默认页面：WooCommerce 的 Shop / Cart / Checkout 页面必须存在，Elementor 模板是覆盖它们的展示层
  • 优惠券区域：确保自定义结账页保留了优惠券输入框，否则用户无法使用折扣码
  • 测试完整购买流程：自定义后必须走一遍完整流程——添加购物车 → 查看购物车 → 结账 → 支付 → 订单完成

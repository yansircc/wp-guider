注册 Google Search Console + 验证站点所有权

Google Analytics L2

📖 What — 这是什么

Google Search Console（GSC）是 Google 免费提供的站长工具，用于监控网站在 Google 搜索中的表现。注册后你可以查看哪些关键词带来了流量、哪些页面被收录、是否有抓取错误等。

使用 GSC 前必须完成站点所有权验证——向 Google 证明你是网站的管理者。WordPress 站点最常用的验证方式是通过 SEO 插件（Yoast / SEOPress）添加 HTML 标签。

🎯 Why — 为什么要做

没有 GSC，你对网站在 Google 搜索中的表现完全是盲的——不知道被收录了多少页面、哪些关键词有排名、是否有技术问题阻碍抓取。

GSC 是 SEO 工作的基础设施，所有后续的搜索优化工作（提交站点地图、监控排名、排查收录问题）都依赖它。

🔧 How — 怎么做

1. 注册 GSC

  1. 访问 search.google.com/search-console
  2. 使用 Google 账号登录
  3. 选择资源类型：
    • 网域（Domain）：覆盖所有子域名，需要 DNS 验证
    • 网址前缀（URL Prefix）：只覆盖指定 URL，验证方式更多更简单（推荐新手使用）
  4. 选择「网址前缀」，输入网站完整 URL（如 https://example.com）

2. 通过 SEO 插件验证所有权

  1. 在 GSC 验证页面，展开 HTML 标记 验证方式
  2. 复制 meta 标签中 content 的值（一串字母数字）
  3. 回到 WordPress 后台：
    • Yoast SEO：进入 Yoast SEO → 常规 → 站长工具，粘贴到「Google 验证代码」字段
    • SEOPress：进入 SEOPress → 高级 → Google Search Console，粘贴验证代码
  4. 保存设置
  5. 回到 GSC 页面，点击 验证
验证成功后不要删除 SEO 插件中的验证代码。Google 会定期重新检查，删除后可能导致验证失效。

⚠️ 常见问题与注意事项

  • http vs https：注册时 URL 必须与网站实际使用的协议一致。如果网站是 https，就输入 https 开头的地址
  • www vs 非 www：www.example.com 和 example.com 在 GSC 中是两个不同的资源。建议用「网域」类型或两个都注册
  • 验证失败：最常见的原因是缓存——SEO 插件保存后前台还是旧页面。清除缓存后重试
  • 数据不会立即出现：GSC 需要 24-48 小时才会开始显示数据，新注册后请耐心等待
  • 本地开发环境无法验证：GSC 需要 Google 能访问到你的网站，LocalWP 等本地环境无法完成验证

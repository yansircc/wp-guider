注册 GA4 + 安装追踪代码（通过 SEO 插件）

Google Analytics L2

📖 What — 这是什么

Google Analytics 4（GA4）是 Google 的网站流量分析工具，用于追踪访客行为：有多少人访问了网站、来自哪里、看了哪些页面、停留了多久。

GA4 通过在网站中嵌入一段 JavaScript 追踪代码来收集数据。在 WordPress 中，最简单的安装方式是通过 SEO 插件（SEOPress）直接配置追踪 ID，无需手动修改代码。

🎯 Why — 为什么要做

没有流量分析，你无法回答最基本的业务问题：网站有没有人看？哪个页面最受欢迎？访客从哪里来？推广投入有没有带来效果？

GA4 是做数据驱动决策的基础——从"我觉得"变成"数据显示"。

🔧 How — 怎么做

1. 注册 GA4

  1. 访问 analytics.google.com，用 Google 账号登录
  2. 点击 开始衡量（Start measuring）
  3. 填写账号名称（如公司名）
  4. 创建媒体资源（Property）：填写网站名称，选择时区和货币
  5. 填写业务信息，点击创建
  6. 选择平台 网站（Web）
  7. 输入网站 URL 和数据流名称，点击 创建数据流
  8. 记录 衡量 ID（格式为 G-XXXXXXXXXX）

2. 通过 SEOPress 安装追踪代码

  1. 进入 WordPress 后台 SEOPress → 高级
  2. 找到 Google Analytics 部分
  3. 在「追踪 ID」字段粘贴你的 G-XXXXXXXXXX ID
  4. 保存设置

如果使用 Yoast SEO（免费版不支持 GA），可以安装 Insert Headers and Footers 插件，将 GA4 的 gtag.js 代码片段粘贴到 Header 区域。

3. 验证安装

  1. 打开网站前台，在新标签页打开 GA4 后台
  2. 进入 报告 → 实时（Realtime）
  3. 如果看到有 1 个活跃用户（就是你自己），说明安装成功
实时报告有 30 秒左右的延迟，刷新几次如果看到数据就代表安装正确。

⚠️ 常见问题与注意事项

  • 衡量 ID 格式：GA4 的 ID 以 G- 开头。如果看到 UA- 开头，那是旧版 Universal Analytics（已停用）
  • 安装后没有数据：最常见的原因是浏览器安装了广告拦截插件（如 uBlock Origin），它会阻止 GA 脚本加载。用无痕模式测试
  • 不要安装多次：如果通过 SEOPress 安装了 GA，就不要再用其他插件或手动添加。重复安装会导致数据翻倍
  • 本地环境也能测试：GA4 追踪代码在本地环境也能工作（只要浏览器能访问 Google 服务器），但本地环境的数据会混入正式数据，注意区分
  • 数据保留期：GA4 默认数据保留 2 个月，建议在 管理 → 数据设置 → 数据保留 中改为 14 个月

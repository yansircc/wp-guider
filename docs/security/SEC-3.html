隐藏登录 URL

security L3

📖 What — 这是什么

WPS Hide Login 是一个轻量级 WordPress 插件，可以将默认的登录地址 /wp-admin 和 /wp-login.php 替换为你自定义的路径。安装后，原有的登录入口会返回 404，只有知道新地址的人才能访问登录页面。

这种做法属于 Security through Obscurity（隐蔽式安全）。它不是万能的防护手段，但作为整体安全策略的一层，能有效减少自动化攻击的噪音。

🎯 Why — 为什么要做

互联网上大量的自动化攻击脚本会扫描 /wp-login.php 并发起暴力破解。这些机器人不会去猜测自定义路径，因此隐藏默认入口就能过滤掉绝大多数自动化攻击。

好处不仅是安全性：大量失败的登录尝试会消耗服务器资源、撑大日志文件。隐藏登录 URL 后，服务器负载和日志噪音都会明显降低。

隐藏登录 URL 应与强密码、双因素认证（2FA）、Wordfence 等措施配合使用，不要作为唯一的防护手段。

🔧 How — 怎么做

1. 安装并激活插件

locwp wp wp-train -- plugin install wps-hide-login --activate

2. 配置自定义登录路径

  1. 登录后台，进入 设置 → WPS Hide Login
  2. 在 Login url 字段中输入自定义路径，例如 my-secret-login
  3. 设置 Redirection url 为 404（访问原 wp-admin 时显示 404 页面）
  4. 点击保存更改

3. 验证配置

  • 访问 http://your-site.local/my-secret-login — 应正常显示登录页
  • 访问 http://your-site.local/wp-admin — 应返回 404
  • 访问 http://your-site.local/wp-login.php — 应返回 404

4. 忘记自定义 URL 的恢复方法

如果忘记了新的登录地址，有两种恢复方式：

# 方法一：WP-CLI 直接停用插件
locwp wp wp-train -- plugin deactivate wps-hide-login

# 方法二：通过数据库查看当前设置
locwp wp wp-train -- option get whl_page

停用插件后，默认的 /wp-login.php 会立即恢复。

⚠️ 常见问题与注意事项

  • 务必记住自定义 URL，建议用密码管理器保存。忘记 URL 等于锁了自己的门
  • 自定义路径不要用常见词（如 login、admin、signin），否则还是容易被猜到
  • 部分缓存插件可能缓存 404 页面，导致新旧 URL 行为异常。修改后清除缓存再测试
  • 如果同时使用了安全插件（如 Wordfence），确认两者的登录保护规则不冲突
  • WPS Hide Login 只修改了前端入口，WP-CLI 和 REST API 的认证不受影响

综合故障注入 — 白屏（WSOD）

troubleshooting L5

📖 What — 这是什么

White Screen of Death (WSOD) 是 WordPress 最臭名昭著的故障现象：页面一片空白，没有任何错误提示。这通常是 PHP 致命错误导致的，但因为默认关闭了错误显示，所以什么都看不到。

在故障注入训练中，系统会故意制造一个白屏故障，你需要用系统性方法诊断并修复。

🎯 Why — 为什么要做

白屏是 WordPress 运维中最高频的紧急故障。可能的原因包括：

  • 插件/主题的 PHP 致命错误
  • PHP 内存耗尽
  • wp-config.php 语法错误
  • 数据库连接失败

掌握系统性的白屏排障流程，才能在真实事故中冷静应对。

🔧 How — 怎么做

系统性排障流程

  1. 开启调试 — 获取错误信息 
    locwp wp wp-train -- config set WP_DEBUG true --raw
locwp wp wp-train -- config set WP_DEBUG_LOG true --raw
  2. 查看日志 — 找到错误线索 
    tail -30 wp-content/debug.log
  3. 根据日志定位
    • 错误指向插件 → 停用该插件
    • 错误指向主题 → 切换默认主题
    • 内存耗尽 → 增加 WP_MEMORY_LIMIT
    • 数据库错误 → 检查 wp-config.php 中的数据库配置
  4. 如果日志没有线索 
    # 停用所有插件
locwp wp wp-train -- plugin deactivate --all

# 问题消失？→ 逐个启用排查
# 问题仍在？→ 切换默认主题
locwp wp wp-train -- theme activate twentytwentyfour

# 仍然白屏？→ 检查 wp-config.php
php -l wp-config.php
  5. 验证修复 — 访问前后台确认正常

⚠️ 常见问题与注意事项

  • 白屏不一定是 PHP 错误——也可能是 Web 服务器配置问题（nginx/Apache）
  • WordPress 5.2+ 有"恢复模式"：致命错误时会发邮件给管理员，提供恢复链接
  • 如果 WP-CLI 也无法执行（PHP 级别崩溃），说明问题在 wp-config.php 或 PHP 环境本身
  • 排障全程保持记录：每一步做了什么、结果是什么。这有助于复盘和学习
  • 修复后别忘了关闭调试模式，恢复正常配置

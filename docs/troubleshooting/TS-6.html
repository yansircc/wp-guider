functions.php 语法错误修复

troubleshooting L5

📖 What — 这是什么

functions.php 是 WordPress 主题中的核心文件，相当于"主题专属的插件"。它在每次页面加载时执行，用于注册菜单、加载样式、添加自定义功能等。

由于 functions.php 是 PHP 代码，任何语法错误（缺少分号、括号不匹配、拼写错误）都会导致 WordPress 完全崩溃——白屏或 500 错误。

🎯 Why — 为什么要做

这是 WordPress 排障中的经典场景：用户在后台"主题编辑器"中编辑了 functions.php，保存后整个站点白屏，后台也进不去了。

知道如何从文件系统层面修复 PHP 语法错误，是 WordPress 运维的救命技能。

🔧 How — 怎么做

1. 定位错误

# 查看 debug.log
tail -20 wp-content/debug.log

典型的语法错误日志：

[30-Mar-2026 10:30:00 UTC] PHP Parse error:
  syntax error, unexpected '}', expecting ';'
  in /app/public/wp-content/themes/astra/functions.php on line 58

2. 用 PHP 检查语法

# 检查文件是否有语法错误
php -l wp-content/themes/astra/functions.php

输出 No syntax errors detected 表示语法正确。

3. 修复错误

根据错误提示定位到具体行号，常见的语法错误：

  • 缺少分号 — echo "hello" → echo "hello";
  • 括号不匹配 — 检查 { 和 } 是否成对
  • 引号不匹配 — 检查字符串中的引号是否闭合
  • 函数名拼写错误 — add_acton → add_action

4. 紧急恢复（无法修复时）

# 方法一：用 WP-CLI 切换主题（绕过 functions.php）
locwp wp wp-train -- theme activate twentytwentyfour

# 方法二：重命名主题目录
mv wp-content/themes/astra wp-content/themes/astra.bak

5. 验证修复

# 语法检查通过
php -l wp-content/themes/astra/functions.php

# 切回主题
locwp wp wp-train -- theme activate astra

# 检查前后台是否正常

⚠️ 常见问题与注意事项

  • 这就是为什么建议启用 DISALLOW_FILE_EDIT — 防止从后台直接编辑 PHP 文件
  • 修改 functions.php 前务必先创建快照或备份
  • 如果是第三方主题，不要直接修改 functions.php——使用子主题，否则主题更新会覆盖你的修改
  • PHP 的错误提示行号可能不完全准确——实际错误可能在提示行的前几行
  • functions.php 文件末尾不要有多余的空行或 ?> 闭合标签，这可能导致"headers already sent"错误

更新插件

备份与维护 L2

📖 What — 这是什么

WordPress 插件需要定期更新，以获取新功能、修复 Bug 和安全补丁。当插件有可用更新时，后台会在「插件」菜单和仪表盘中显示通知标记。

更新插件本质上是用新版本的文件替换旧版本。WordPress 会自动下载、解压并替换插件目录中的文件，然后触发插件的升级脚本（如果有的话）。

🎯 Why — 为什么要做

不更新插件是 WordPress 网站被攻击的首要原因。已知漏洞会被公开披露，攻击者专门扫描运行旧版本插件的网站。

但更新也不是无风险的——新版本可能引入兼容性问题，破坏现有功能。因此更新前备份、更新后验证是标准流程。

🔧 How — 怎么做

安全更新流程

  1. 更新前备份：创建 WP Reset 快照或 UpdraftPlus 备份
  2. 进入 插件 → 已安装的插件
  3. 有更新的插件顶部会出现黄色提示条
  4. 点击提示中的 现在更新（update now）链接
  5. 等待更新完成（显示「已更新」）
  6. 更新后验证：检查网站前台和相关功能是否正常

批量更新

  1. 进入 仪表盘 → 更新
  2. 在「插件」区域勾选要更新的插件
  3. 点击 更新插件
逐个更新还是批量更新？逐个更新更安全——如果出问题可以精确定位是哪个插件导致的。批量更新更快但出问题时排查困难。

⚠️ 常见问题与注意事项

  • 更新前必须备份：这是铁律。如果更新导致白屏，可以通过恢复快照回退
  • 更新失败/卡住：可能是服务器超时或磁盘空间不足。不要刷新页面，等待完成或通过 FTP 手动替换插件文件
  • 更新后白屏：可能是插件与当前 WordPress 版本或其他插件不兼容。通过 FTP 将问题插件的目录重命名来禁用它
  • 自动更新：WordPress 支持为每个插件单独开启自动更新。在插件列表中点击「启用自动更新」链接即可。对于关键站点，建议只对安全类插件开启自动更新
  • 付费插件：部分付费插件需要有效的许可证才能接收更新通知

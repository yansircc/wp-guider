All-in-One Migration 跨服务器迁移

备份与维护 L3 NEW

📖 What — 这是什么

All-in-One WP Migration 是一个站点迁移插件，可以将整个 WordPress 站点打包为单个 .wpress 文件，包括数据库、媒体、插件、主题——所有内容。

在目标服务器上安装同一个插件，导入这个文件即可完成迁移。它会自动处理 URL 替换、序列化数据修复等复杂问题。

🎯 Why — 为什么要做

站点迁移是 WordPress 运维中最常见也最容易出错的操作之一。手动迁移需要：导出数据库、修改 URL、上传文件、修复序列化数据——每一步都可能出错。

All-in-One WP Migration 把这些步骤封装成了"导出 → 导入"两步操作，大幅降低了迁移失败的风险。

🔧 How — 怎么做

1. 在源站点导出

# 安装插件
locwp wp wp-train -- plugin install all-in-one-wp-migration --activate
  1. 进入后台 All-in-One WP Migration → Export
  2. 点击 Export To → File
  3. 等待打包完成，下载 .wpress 文件

2. 在目标站点导入

  1. 在目标 WordPress 上安装并激活 All-in-One WP Migration
  2. 进入 All-in-One WP Migration → Import
  3. 上传或拖拽 .wpress 文件
  4. 确认导入（会覆盖目标站点的所有数据）
  5. 导入完成后，用 源站点的管理员账号 重新登录

3. 导入后检查

  • 刷新固定链接：设置 → 固定链接 → 保存
  • 检查媒体文件是否正常显示
  • 检查表单、联系方式等功能是否正常
免费版有导入文件大小限制（通常 512MB）。对于大型站点，可以使用付费扩展或在导出时排除媒体文件（单独迁移 uploads 目录）。

⚠️ 常见问题与注意事项

  • 导入会完全覆盖目标站点，包括管理员账号和密码
  • 导入后必须用源站点的登录凭据登录
  • 如果文件过大导致上传失败，检查 PHP 的 upload_max_filesize 和 post_max_size 设置
  • 迁移后如果遇到 CSS/图片加载问题，通常是 URL 替换不完整，用 WP-CLI 的 search-replace 补充替换
  • All-in-One 适合一次性迁移；持续备份用 UpdraftPlus，快速回滚用 WP Reset

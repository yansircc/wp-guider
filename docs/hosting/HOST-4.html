开启 SSH 访问权限

主机 L3

📖 What — 这是什么

SSH（Secure Shell）提供加密的远程终端访问，允许你通过命令行直接操作主机服务器。在 Hostinger 上，SSH 需要手动开启才能使用。

开启后可以通过 SSH 客户端（如 Terminal、PuTTY）连接到服务器，执行命令行操作。

🎯 Why — 为什么要做

SSH 访问对于高级操作至关重要：使用 WP-CLI 批量管理 WordPress、直接编辑配置文件、排查服务器层面的问题、执行数据库操作等。

在 WP Guider 培训中，部分高级任务（如 wp-cli 操作、文件权限修改）需要 SSH 访问。

🔧 How — 怎么做

  1. 登录 Hostinger hPanel
  2. 选择目标网站
  3. 进入 Advanced → SSH Access
  4. 点击 Enable（启用 SSH）
  5. 页面会显示连接信息：
    • Host：SSH 服务器地址
    • Port：SSH 端口（通常是 65002 而非默认 22）
    • Username：SSH 用户名
  6. 密码使用你的 Hostinger 账号密码

连接测试

ssh -p 65002 username@ssh-host.hostinger.com

⚠️ 常见问题与注意事项

  • SSH 仅在 Business 及以上方案中可用，Premium/Single 方案不支持
  • Hostinger 使用非标准端口（如 65002），连接时必须指定 -p 参数
  • 共享主机的 SSH 权限有限，不是 root 权限，无法执行 sudo 命令
  • 如果连接超时，检查本地网络是否屏蔽了该端口
  • 登录后的工作目录就是网站根目录，WordPress 文件就在当前目录下

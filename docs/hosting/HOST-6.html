生成 SSH 密钥对 + 免密登录

主机 L4

📖 What — 这是什么

SSH 密钥对由公钥和私钥组成，用于替代密码进行身份验证。将公钥放在服务器上，本地保留私钥，连接时自动验证，无需输入密码。

这是 SSH 最安全也最便捷的认证方式，比密码登录更难被暴力破解。

🎯 Why — 为什么要做

每次 SSH 连接都输入密码既麻烦又不安全（密码可能被截获）。配置密钥后一键连接，同时安全性大幅提升。对于需要频繁 SSH 操作的教练来说，这是效率刚需。

自动化脚本和 CI/CD 流程也依赖密钥认证。

🔧 How — 怎么做

第一步：生成密钥对（本地执行）

ssh-keygen -t ed25519 -C "your_email@example.com"
  • 按 Enter 接受默认路径（~/.ssh/id_ed25519）
  • 设置密码短语（passphrase）或留空
  • 生成两个文件：id_ed25519（私钥）和 id_ed25519.pub（公钥）

第二步：上传公钥到 Hostinger

  1. 登录 Hostinger hPanel → Advanced → SSH Access
  2. 找到 SSH Keys 区域
  3. 点击 Add SSH Key
  4. 将 id_ed25519.pub 的内容粘贴进去
  5. 保存

第三步：测试连接

ssh -p 65002 username@ssh-host.hostinger.com

如果配置正确，应该直接登录而不需要输入密码。

⚠️ 常见问题与注意事项

  • 私钥文件（id_ed25519）绝对不能分享给任何人，泄露等于泄露了登录权限
  • 如果有多台服务器，可以为每台生成不同的密钥对，或使用同一个公钥
  • 私钥文件权限必须是 600：chmod 600 ~/.ssh/id_ed25519
  • 推荐使用 ed25519 算法，比传统 RSA 更安全且密钥更短
  • 如果连接仍然要求密码，检查公钥是否正确上传、文件权限是否正确
  • 可配置 ~/.ssh/config 简化连接命令，避免每次输入端口和用户名
